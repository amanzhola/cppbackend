# 📘 Lesson 14 — boost::asio::post

## 📌 Тема

Асинхронное выполнение функций через `boost::asio::post` и поведение задач внутри очереди `io_context` при одном и нескольких потоках исполнения.

---

# 🎯 Цель урока

| 🎯 Концепция работы boost::asio::post в разных сценариях выполнения | 📖 Подробное объяснение того, как задачи ставятся в очередь и когда именно они начинают исполняться в зависимости от контекста выполнения программы | ⚙️ Как это проявляется в коде через io_context, post и run | 🧠 Почему это критично для понимания асинхронного программирования и многопоточности | 🔥 Что показывает практика выполнения демо-примеров | ✅ Итог       |
| ------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- | ------------------------------------------------------------------------------------ | --------------------------------------------------- | ------------ |
| Очередь задач                                                       | Все вызовы post не выполняют код сразу, а помещают его в очередь внутри io_context, откуда он будет извлечён позже                                  | post + io.run                                              | Управление временем выполнения                                                       | Очередь управляет порядком                          | База async   |
| Поток исполнения                                                    | Код выполняется тем потоком, который вызвал io.run, а не тем, который вызвал post                                                                   | thread id                                                  | Важно для дебага                                                                     | Видно в логах                                       | Контроль     |
| Параллельность                                                      | При нескольких потоках задачи могут выполняться одновременно и порядок может меняться                                                               | несколько io.run                                           | Появляется concurrency                                                               | Нет гарантии порядка                                | Риск         |
| strand                                                              | Позволяет сериализовать выполнение задач внутри одной логической группы и гарантирует отсутствие пересечений                                        | make_strand                                                | Защита от гонок                                                                      | Порядок фиксируется                                 | Безопасность |

# 2️⃣ demo2 — выполнение в другом потоке


| ------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------ | ----------------------------------------------------- | ------------------------------------- |
| Разделение потоков                                                                                     | main добавляет задачи, worker выполняет                                                          | producer / consumer модель                                               | контроль потоков                                      | задачи выполняются не там где созданы |

# 🧠 Как формируется порядок (demo5)

| 🧠 Шаг выполнения программы и изменения очереди задач | 📖 Подробное описание того, как меняется очередь задач после каждого выполненного обработчика | 🔥 Что получается в итоге |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------------- | ------------------------- |
| 1                                                     | выполняется Watch movie до запуска io.run                                                     | сразу                     |
| 2                                                     | очередь: Read book → Play football                                                            | FIFO                      |
| 3                                                     | выполняется Read book                                                                         | добавляет Write essay     |
| 4                                                     | очередь: Play football → Write essay                                                          | обновление                |
| 5                                                     | выполняется Play football                                                                     | дальше                    |
| 6                                                     | выполняется Write essay                                                                       | дальше                    |
| 7                                                     | после завершения io.run выполняется Study English                                             | финал                     |

# 🔥 Итог урока

| 🔥 Основная идея урока и выводы по всем демонстрациям использования boost::asio::post и io_context | 📖 Что это означает с точки зрения архитектуры асинхронных программ и многопоточности | 🧠 Главное, что нужно запомнить | ✅ Итог    |
| -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- | ------------------------------- | --------- |
| post                                                                                               | добавляет задачи в очередь                                                            | не выполняет сразу              | async     |
| io.run                                                                                             | выполняет задачи                                                                      | поток определяет выполнение     | executor  |
| multi-thread                                                                                       | ускоряет выполнение                                                                   | но ломает порядок               | риск      |
| strand                                                                                             | восстанавливает порядок                                                               | защищает от гонок               | правильно |

---

# 💡 Ключевая идея

| 💡 Концепция | 📖 Объяснение                                                   |
| ------------ | --------------------------------------------------------------- |
| post         | это добавление задачи в очередь, а не её немедленное выполнение |
| io_context   | это механизм исполнения очереди задач                           |
| потоки       | определяют кто именно будет выполнять задачи                    |
| strand       | обеспечивает безопасный порядок выполнения                      |
