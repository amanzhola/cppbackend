# 🚀 Практическое занятие №8

## Научиться видеть жизненный цикл сессии

---

# 📌 Кратко

| 📌 Что это                                                           | 📖 Полное описание                                                                                                                                                                                                                                                                     |
| -------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Учебный HTTP-сервер с разделением уровней процесса, сессии и запроса | В этом задании сервер показывает, что внешний цикл в `main()` отвечает за новых клиентов, внутренний цикл в `HandleConnection()` отвечает за запросы уже подключённого клиента, а значит серверный процесс, TCP-сессия и HTTP-запрос являются тремя разными уровнями работы приложения |

---

# 🎯 Цель

| 🎯 Что нужно понять | 📖 Что это означает на практике                                                      |
| ------------------- | ------------------------------------------------------------------------------------ |
| Уровень процесса    | Сервер живёт постоянно и не завершается после одного клиента                         |
| Уровень сессии      | Один клиент удерживает одно TCP-соединение в течение своей сессии                    |
| Уровень запроса     | Внутри одной сессии клиент может отправить один или несколько HTTP-запросов          |
| Разделение уровней  | Внешний цикл и внутренний цикл решают разные задачи и не должны путаться между собой |

---

# ⚙️ Основной код

```cpp
while (true) {
    tcp::socket socket(ioc);
    acceptor.accept(socket);
    HandleConnection(socket);
}
```

```cpp
beast::flat_buffer buffer;

while (true) {
    auto request = ReadRequest(socket, buffer);
    if (!request) {
        break;
    }

    auto response = MakeResponse(*request);
    http::write(socket, response);

    if (!response.keep_alive()) {
        break;
    }
}
```

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                                             | 📖 Смысл                                                                                                                                                |
| -------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `main() принимает новых клиентов, HandleConnection() читает запросы текущего клиента, buffer живёт всю сессию` | Серверный процесс, клиентская сессия и отдельные HTTP-запросы существуют на разных уровнях, и именно это нужно научиться видеть в архитектуре программы |

---

# 🌐 Что именно проверяется

| 🌐 Уровень | 📖 Что происходит                                                                         |
| ---------- | ----------------------------------------------------------------------------------------- |
| Процесс    | Программа сервера остаётся запущенной постоянно                                           |
| Сессия     | Один клиент подключается, держит соединение и завершает свою сессию                       |
| Запрос     | В рамках одной сессии клиент присылает HTTP-запросы, которые сервер читает и обрабатывает |
| Буфер      | Один buffer используется на всю сессию, чтобы не потерять данные между запросами          |

| 🔍 Что выполнить                                                                                                            | 📥 Какой результат должен быть                                                                                                                                                  |
| --------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить GET для `/` и `/hello`, затем GET для неизвестного пути и POST для `/hello`, после чего посмотреть серверные логи | Сервер остаётся запущенным, показывает подключение клиента, читает запросы внутри сессии, возвращает правильные ответы и завершает сессию только после окончания работы клиента |

# 🌐 HTTP-смысл задания

| 🌐 Что происходит               | 📖 Почему это важно                                        |
| ------------------------------- | ---------------------------------------------------------- |
| Сервер принимает нового клиента | Это уровень TCP-сессии                                     |
| Сервер читает запросы клиента   | Это уже уровень HTTP-запросов внутри соединения            |
| Сервер остаётся работать дальше | Это уровень жизни серверного процесса                      |
| Буфер живёт на всю сессию       | Это позволяет не терять данные между несколькими запросами |

# ✅ Формулировка проверки

| ✅ Проверка                                                          | ✅ Условие выполнено                                                                                                                                      |
| ------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Представить работу сервера как три уровня: процесс, сессия и запрос | Если видно, что `main()` отвечает за клиентов, `HandleConnection()` отвечает за запросы клиента, а сервер остаётся запущенным постоянно, цель достигнута |

---

# ⚠️ Важно

| ⚠️ Проблема                                  | 📖 Причина и решение                                                                  |
| -------------------------------------------- | ------------------------------------------------------------------------------------- |
| Буфер создаётся внутри цикла чтения запросов | Это ошибка, потому что данные следующего запроса могут уже находиться в старом buffer |
| Сессия путается с жизнью процесса            | Нужно помнить, что отключение клиента не должно завершать весь сервер                 |
| Запрос путается с соединением                | Один клиент может отправить больше одного HTTP-запроса внутри одной сессии            |

# 🏗 Архитектура

| 🏗 Компоненты                                                                          | 📖 Роль каждого компонента                                                                                                                                                                                                 |
| -------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `main`, `acceptor.accept`, `HandleConnection`, `ReadRequest`, `buffer`, `MakeResponse` | `main` управляет жизнью процесса и принимает клиентов, `HandleConnection` управляет клиентской сессией, `ReadRequest` читает HTTP-запросы, buffer сохраняет данные на всю сессию, а `MakeResponse` формирует ответ клиенту |
