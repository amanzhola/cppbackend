# 🚀 Практическое занятие №1

## Сервер, который не умирает после первого клиента

---

# 📌 Кратко

| 📌 Что это                                                                                | 📖 Полное описание                                                                                                                                                                                                                                                                                      |
| ----------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Практическая работа по серверу, который не завершается после обслуживания первого клиента | В этой задаче нужно показать, что сервер живёт дольше одного соединения, постоянно принимает новые подключения через цикл `while (true)`, использует `acceptor.accept(socket)` для блокирующего ожидания клиента и после завершения одного сеанса продолжает работать дальше без перезапуска приложения |

---

# 🎯 Цель

| 🎯 Что нужно понять                                                         | 📖 Что это означает на практике                                                                                                                                                                                                             |
| --------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Разницу между одним клиентским сеансом и жизнью всего серверного приложения | Один запрос или один клиент не должен завершать процесс сервера; сервер обязан оставаться запущенным, снова ждать новое подключение, снова вызывать обработку соединения и отвечать на последующие запросы столько раз, сколько потребуется |

---

# ⚙️ Основной код

```cpp id="code103main"
int main() {
    net::io_context ioc;

    const auto address = net::ip::make_address("0.0.0.0");
    constexpr unsigned short port = 8080;

    tcp::acceptor acceptor(ioc, {address, port});

    while (true) {
        tcp::socket socket(ioc);
        acceptor.accept(socket);
        HandleConnection(socket);
    }
}
```

---

# 🧠 Что здесь изучается

| 🧠 Концепция                                   | 📖 Объяснение                                                                                                                                                                                                                                                                              |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Цикл приёма подключений `while (true)`         | Сервер не ограничивается одним клиентом; после завершения обработки текущего соединения создаётся новый `tcp::socket`, выполняется новый `acceptor.accept(socket)`, затем снова вызывается `HandleConnection(socket)`, и приложение продолжает работу без завершения после первого запроса |
| Блокирующая операция `acceptor.accept(socket)` | Эта функция останавливает выполнение в текущем месте и ждёт, пока подключится новый клиент; когда клиент приходит, сокет заполняется подключением, после чего управление возвращается в программу и начинается обработка                                                                   |
| Разделение жизни клиента и жизни сервера       | Клиент может подключиться, получить ответ и отключиться, но это не означает завершение сервера; сервер существует независимо от одного конкретного подключения и продолжает принимать следующих клиентов                                                                                   |
| Вызов `HandleConnection(socket)`               | В этом месте происходит обработка одного конкретного соединения: чтение запроса, формирование ответа и завершение сеанса; после этого сервер не умирает, а возвращается к ожиданию следующего подключения                                                                                  |

| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

# 🖥 Поведение сервера

| 🖥 Что происходит внутри сервера                                                                                                                                                                                                                                                          | 🌐 Что это означает снаружи                                                                                                                                                               |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Server started on 0.0.0.0:8080, acceptor waits for client connection, client connected, HandleConnection(socket) called, request processed, client disconnected, loop continues, acceptor waits for next client connection, client connected again, HandleConnection(socket) called again | Первый клиент получает ответ, сервер остаётся запущенным, второй клиент тоже получает ответ, третий клиент тоже получает ответ, и приложение не нужно перезапускать после каждого запроса |

| ------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Клиент может несколько раз подряд выполнить `curl http://localhost:8080/hello`, и сервер будет отвечать каждый раз | Это означает, что приложение реализует не одноразовый сценарий обработки соединения, а настоящий серверный цикл, где после завершения одного сеанса программа возвращается к ожиданию следующего подключения вместо завершения процесса |

# ⚠️ Что будет, если убрать цикл

| ❌ Неправильный вариант                                                                                           | 💥 Что сломается                                                                                                                                                                             |
| ---------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `tcp::socket socket(ioc); acceptor.accept(socket); HandleConnection(socket);` без внешнего цикла приёма клиентов | Сервер примет только одно подключение, обработает только одного клиента, после выхода из `main()` завершит работу и перестанет отвечать на последующие запросы без нового запуска приложения |

# 🏗 Архитектура

| 🏗 Компоненты                                                                 | 📖 Роль каждого компонента                                                                                                                                                                                                                                                                                                           |
| ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `net::io_context`, `tcp::acceptor`, `tcp::socket`, `HandleConnection(socket)` | `net::io_context` обслуживает операции ввода-вывода; `tcp::acceptor` слушает адрес `0.0.0.0:8080` и принимает входящие подключения; `tcp::socket` представляет одно клиентское соединение; `HandleConnection(socket)` обрабатывает конкретный запрос клиента, после чего цикл сервера возвращается к ожиданию следующего подключения |
