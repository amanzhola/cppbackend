# 🚀 Практическое занятие №2

## Несколько запросов в одной HTTP-сессии

---

# 📌 Кратко

| 📌 Что это                                                                                          | 📌 Описание                                                                                                                                                                                                                     |
| --------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Практическое задание про повторное использование одного TCP-соединения для нескольких HTTP-запросов | В этой работе нужно понять, что внешний цикл сервера отвечает за новых клиентов, а внутренний цикл внутри `HandleConnection(socket)` отвечает за несколько HTTP-запросов от одного и того же клиента в рамках одной HTTP-сессии |

---

# 🎯 Цель

| 🎯 Что нужно понять                                     | 📖 Что это означает на практике                                                                                                                                                                                                                 |
| ------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Одно TCP-соединение может нести несколько HTTP-запросов | Клиент подключается один раз, сервер принимает это подключение один раз, но затем внутри текущего соединения может прочитать, обработать и отправить ответы на несколько запросов подряд, не создавая новое TCP-подключение для каждого запроса |

---

# 💡 Идея урока

| 💡 Внешний и внутренний циклы                                                                                                                                                             | 📖 Смысл                                                                                                                                                                         |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `cpp<br>while (true) {<br>&nbsp;&nbsp;&nbsp;&nbsp;tcp::socket socket(ioc);<br>&nbsp;&nbsp;&nbsp;&nbsp;acceptor.accept(socket);<br>&nbsp;&nbsp;&nbsp;&nbsp;HandleConnection(socket);<br>}` | Внешний цикл отвечает за новых клиентов: сервер ждёт подключение, принимает клиента и передаёт сокет в обработчик соединения                                                     |
| `cpp<br>while (true) {<br>&nbsp;&nbsp;&nbsp;&nbsp;http::read(...);<br>&nbsp;&nbsp;&nbsp;&nbsp;http::write(...);<br>}`                                                                     | Внутренний цикл отвечает за новые запросы в рамках уже открытого соединения: пока клиент не закрыл соединение, сервер может читать и обрабатывать несколько HTTP-запросов подряд |

---

# 🧠 Главный интеллектуальный результат урока

| 🧠 Что важно понять                                                                                     | 📖 Почему это главное                                                                                                                                                              |
| ------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Внешний цикл — это новые подключения, внутренний цикл — это новые запросы в рамках текущего подключения | Это базовая серверная модель: сервер живёт долго, принимает клиентов по одному, а каждый клиент может отправить не один запрос, а несколько, пока TCP-соединение остаётся открытым |

# ⚙️ Основной код

```cpp
int main() {
    try {
        net::io_context ioc;

        const auto address = net::ip::make_address("0.0.0.0");
        constexpr unsigned short port = 8080;

        tcp::acceptor acceptor(ioc, {address, port});

        std::cout << "Server started on 0.0.0.0:8080\n";

        while (true) {
            tcp::socket socket(ioc);
            acceptor.accept(socket);

            std::cout << "Client connected\n";
            HandleConnection(socket);
        }
    } catch (const std::exception& e) {
        std::cerr << "Fatal error: " << e.what() << '\n';
        return 1;
    }
}
```

# 🔄 Внутренний цикл обработки запросов

```cpp
void HandleConnection(tcp::socket& socket) {
    beast::flat_buffer buffer;

    while (true) {
        http::request<http::string_body> req;
        beast::error_code ec;

        http::read(socket, buffer, req, ec);

        if (ec == http::error::end_of_stream) {
            std::cout << "Client disconnected\n";
            break;
        }

        if (ec) {
            std::cerr << "Read error: " << ec.message() << '\n';
            break;
        }

        std::cout << "Request received: " << req.method_string()
                  << " " << req.target() << '\n';

        HandleRequest(req, socket);

        if (!req.keep_alive()) {
            std::cout << "Connection will be closed by client\n";
            break;
        }
    }

    beast::error_code ec;
    socket.shutdown(tcp::socket::shutdown_send, ec);
}
```

# ⚠️ Что было бы без внутреннего цикла

| ❌ Неправильный вариант                                                                                          | 💥 Что сломается                                                                                                                                                                       |
| --------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `cpp<br>http::request<http::string_body> req;<br>http::read(socket, buffer, req);<br>http::write(socket, res);` | Сервер обработает только один HTTP-запрос из текущего TCP-соединения, после чего завершит обслуживание этого клиента и не сможет показать идею нескольких запросов в одной HTTP-сессии |

| 🏗 Компоненты                                                                                                        | 📖 Роль каждого компонента                                                                                                                                                                                                                                                                                                            |
| -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `net::io_context`, `tcp::acceptor`, `tcp::socket`, `HandleConnection(socket)`, `http::read(...)`, `http::write(...)` | `tcp::acceptor` принимает новых клиентов во внешнем цикле; `tcp::socket` представляет текущее соединение; `HandleConnection(socket)` запускает внутренний цикл обработки запросов; `http::read(...)` читает очередной HTTP-запрос из уже открытого TCP-соединения; `http::write(...)` отправляет очередной HTTP-ответ тому же клиенту |
