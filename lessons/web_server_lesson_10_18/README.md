# 🚀 Практическое занятие №17

## Что именно делает `std::move(socket)`

---

# 📌 Краткое описание

| 📌 Что изучается                                      | 📖 Полное описание                                                                                                                                                                                                                                                                                          |
| ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Передача клиентского сокета в поток через `std::move` | В этом занятии разбирается, почему объект `tcp::socket` нельзя копировать, почему передача сокета в новый поток без `std::move` обычно приводит к ошибке компиляции и почему правильная модель состоит в том, чтобы после `accept` переместить сокет в поток, который будет обслуживать конкретного клиента |

---

# 🎯 Цель

| 🎯 Что нужно понять                     | 📖 Что это означает на практике                                                                                                                                                                            |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Сокет нельзя копировать                 | `tcp::socket` владеет системным ресурсом операционной системы, поэтому копирование для него запрещено и попытка передать его в поток как обычный копируемый объект обычно заканчивается ошибкой компиляции |
| Сокет нужно перемещать                  | `std::move(socket)` сообщает, что объект можно передать дальше как перемещаемый ресурс, после чего владение соединением переходит в новый контекст выполнения                                              |
| Поток должен получить собственный сокет | После принятия подключения главный поток должен передать клиентский сокет в поток обработки, чтобы этот поток независимо читал запросы и отправлял ответы именно этому клиенту                             |

---

# ⚙️ Основной код

```cpp
std::thread t(
    [](tcp::socket socket) {
        HandleConnection(std::move(socket));
    },
    std::move(socket));

t.detach();
```

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                         | 📖 Смысл                                                                                                                                                                                                              |
| -------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `accept → получить socket → std::move(socket) → передать в поток → detach` | После принятия клиента сервер не пытается копировать сокет, а перемещает его в новый поток обработки, потому что сокет является ресурсом с владением, который должен обслуживаться тем кодом, которому он был передан |

---

# 🔍 Что будет без `std::move`

| 🔍 Ситуация                                         | 📖 Что произойдёт                                                                                                                                                           |
| --------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Передать `socket` в `std::thread` без `std::move`   | Поток попытается получить аргумент через копирование, но `tcp::socket` не поддерживает копирование, поэтому такой код обычно не компилируется                               |
| Передать `socket` в `std::thread` через `std::move` | Сокет будет корректно перемещён в новый поток, владение соединением перейдёт обработчику клиента, и код будет соответствовать правильной модели работы с системным ресурсом |

---

# 🌐 Что происходит в программе

| 🌐 Шаг                                | 📖 Объяснение                                                                                                                                   |
| ------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `acceptor.accept(socket)`             | Сервер принимает новое клиентское подключение и получает рабочий объект сокета, связанный именно с этим клиентом                                |
| `std::move(socket)`                   | Главный поток передаёт сокет дальше как перемещаемый объект, показывая, что текущий объект больше не должен использоваться как прежний владелец |
| `std::thread(...)`                    | Создаётся новый поток, которому передаётся сокет для независимой обработки конкретного клиента                                                  |
| `HandleConnection(std::move(socket))` | Поток получает сокет и работает с этим соединением, читает HTTP-запросы, формирует ответы и завершает соединение при необходимости              |
| `detach()`                            | Главный поток не ждёт завершения клиентского потока и сразу возвращается к `accept`, чтобы принимать следующие подключения                      |

# 🧪 Проверка

| 🧪 Что проверить                                       | ✅ Какой результат должен быть                                                                                                      |
| ------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| Проект собирается с использованием `std::move(socket)` | Код успешно компилируется, потому что сокет передаётся как перемещаемый ресурс, а не как копируемый объект                         |
| Сервер запускается и отвечает на `curl`                | После запуска сервер принимает подключения и возвращает HTTP-ответ без падения                                                     |
| Несколько клиентов подключаются одновременно           | Сервер продолжает принимать подключения и не блокируется на одном клиенте, потому что соединения обслуживаются в отдельных потоках |

# 🌐 HTTP смысл

| 🌐 Что происходит на уровне HTTP                                                     | 📖 Почему это важно                                                                                                                     |
| ------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| Каждый клиент приходит в сервер через собственное TCP-соединение и собственный сокет | Чтобы поток мог читать запрос и отправлять ответ именно этому клиенту, он должен владеть его сокетом, а не обращаться к случайной копии |
| Сервер читает запрос, формирует ответ и управляет соединением                        | Передача сокета через перемещение делает модель владения соединением явной и правильной для многопоточного сервера                      |
| Один клиент не должен мешать другому                                                 | Когда сокет конкретного клиента передан в отдельный поток, разные клиентские соединения могут обслуживаться независимо друг от друга    |
