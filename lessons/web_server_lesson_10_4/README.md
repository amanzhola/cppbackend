# 🚀 Практическое занятие №2

## Несколько запросов в одной HTTP-сессии

---

# 📌 Кратко

| 📌 Что это                                    | 📖 Полное описание                                                                                                                                                                                                                |
| --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Повторное использование одного TCP-соединения | В этой работе сервер показывает, что одно TCP-соединение может использоваться для нескольких HTTP-запросов подряд, потому что внешний цикл обрабатывает клиентов, а внутренний цикл обрабатывает запросы внутри одного соединения |

---

# 🎯 Цель

| 🎯 Что нужно понять                                     | 📖 Что это означает на практике                                                                                                                                         |
| ------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Одно TCP-соединение может нести несколько HTTP-запросов | Клиент подключается один раз, сервер принимает соединение один раз, но затем может читать и обрабатывать несколько HTTP-запросов подряд без создания нового подключения |

---

# 💡 Главная идея

| 🧠 Ключевая логика                                                                                                         | 📖 Смысл                                                                                      |
| -------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| `while (true) { tcp::socket socket(ioc); acceptor.accept(socket); HandleConnection(socket); } // внешний цикл по клиентам` | Внешний цикл отвечает за новые подключения клиентов и передаёт каждое соединение в обработчик |
| `while (true) { http::read(...); http::write(...); } // внутренний цикл по запросам`                                       | Внутренний цикл обрабатывает несколько HTTP-запросов в рамках одного TCP-соединения           |

---

# 🧠 Главный результат

| 🧠 Что важно понять                                   | 📖 Почему это важно                                                                                                                 |
| ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| Внешний цикл это клиенты, внутренний цикл это запросы | Сервер должен жить долго и обслуживать много клиентов, при этом каждый клиент может отправить несколько запросов в одном соединении |

---

# ⚙️ Основной код

```cpp
int main() {
    net::io_context ioc;

    const auto address = net::ip::make_address("0.0.0.0");
    constexpr unsigned short port = 8080;

    tcp::acceptor acceptor(ioc, {address, port});

    while (true) {
        tcp::socket socket(ioc);
        acceptor.accept(socket);
        HandleConnection(socket);
    }
}
```

---

# 🔄 Внутренний цикл

```cpp
void HandleConnection(tcp::socket& socket) {
    beast::flat_buffer buffer;

    while (true) {
        http::request<http::string_body> req;
        beast::error_code ec;

        http::read(socket, buffer, req, ec);

        if (ec == http::error::end_of_stream) {
            break;
        }

        if (ec) {
            break;
        }

        HandleRequest(req, socket);

        if (!req.keep_alive()) {
            break;
        }
    }

    socket.shutdown(tcp::socket::shutdown_send);
}
```

# ⚠️ Что будет без внутреннего цикла

| ❌ Ошибка                        | 💥 Последствие                            |
| ------------------------------- | ----------------------------------------- |
| обработка только одного запроса | соединение закрывается сразу              |
| нет цикла чтения                | сервер не поддерживает несколько запросов |
| нет keep-alive                  | клиент вынужден переподключаться          |
