# 🚀 Практическое занятие №8

## Многопоточный HTTP сервер без блокировок

---

# 📌 Кратко

| 📌 Что это                                            | 📖 Полное описание                                                                                                                                                                                                                                                            |
| ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| HTTP сервер с обработкой клиентов в отдельных потоках | В этом задании сервер после каждого `accept` создаёт новый поток, передаёт туда клиентский сокет и сразу возвращается к приёму следующих подключений, поэтому один медленный или занятый клиент больше не блокирует остальных и главный поток не перестаёт выполнять `accept` |

---

# 🎯 Цель

| 🎯 Что нужно понять                                     | 📖 Что это означает на практике                                                                                                                 |
| ------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Один клиент не должен блокировать всех остальных        | Если один клиент обслуживается долго, это должно тормозить только его собственный поток, а не весь сервер целиком                               |
| Главный поток должен продолжать принимать клиентов      | После `accept` сервер не должен застревать в обработке старого соединения, а должен немедленно возвращаться к ожиданию следующего подключения   |
| Обработка клиента может быть вынесена в отдельный поток | Каждый клиент получает собственный поток выполнения, а сервер становится реально многоклиентским и может принимать новые соединения параллельно |

---

# ⚙️ Основной код

```cpp
std::thread(
    [](tcp::socket socket) {
        HandleConnection(std::move(socket));
    },
    std::move(socket)
).detach();
```

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                                                                                                    | 📖 Смысл                                                                                                                                                                             |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `accept → создать поток → передать сокет → detach → сразу вернуться к следующему accept и продолжить приём новых подключений без ожидания завершения старого клиента` | Сервер больше не связывает приём новых клиентов с завершением обработки старого клиента, поэтому исчезает блокировка приёма новых подключений из-за обслуживания уже открытой сессии |

---

# 🌐 Что именно изменилось после переделки сервера

| 🌐 До изменения                                                                                                                                                  | 📖 После изменения                                                                                                                                                     |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Один медленный клиент мог удерживать сервер внутри обработки соединения и тем самым тормозить приём следующих подключений, пока старый клиент не закончит работу | Медленный клиент тормозит только свой поток, а главный поток продолжает выполнять `accept`, принимает новых клиентов и запускает для них отдельные потоки обслуживания |

# 🔍 Проверка и результат

| 🔍 Что выполнить                                                                                                                                                                            | 📥 Какой результат должен быть                                                                                                                                                  |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить несколько одиночных запросов подряд, а затем много параллельных запросов через фоновые `curl` процессы и сразу после этого сделать ещё один отдельный запрос из другого терминала | Сервер отвечает на все запросы, не зависает, не перестаёт принимать подключения и продолжает работать даже тогда, когда несколько клиентов обращаются к нему почти одновременно |

# 🖥 Что видно в логах сервера

| 🖥 Что видно в терминале сервера                                                                                                                                 | 📖 Что это доказывает                                                                                                                                        |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| В логах появляются многочисленные запросы от разных клиентов, а сервер не прекращает принимать новые соединения после первого или десятого обработанного запроса | Главный поток не блокируется обработкой старого клиента, а каждый новый клиент получает собственный поток и обслуживается независимо от остальных соединений |

# 🌐 HTTP и потоковая модель

| 🌐 Что происходит                                                                                                                                             | 📖 Почему это важно                                                                                                                                              |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Клиент подключается к серверу, сервер принимает сокет и немедленно передаёт его в отдельный поток обработки, после чего главный поток возвращается к `accept` | Это позволяет разделить уровень приёма новых подключений и уровень обработки клиентских сессий, что делает сервер ближе к реальной многоклиентской модели работы |
| Один клиент отправляет запросы внутри своей сессии, а другие клиенты могут подключаться параллельно и не ждать окончания чужой обработки                      | Сервер больше не ведёт себя как одноклиентская программа и начинает обслуживать несколько соединений одновременно                                                |

# 🧠 Что это доказывает

| 🧠 Сценарий                                                                                  | 📖 Вывод                                                                                                                                                        |
| -------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Большое количество клиентов подключается одновременно и создаёт сильную нагрузку на сервер   | Сервер выдерживает нагрузку и не превращается в однопоточный блокирующий обработчик одного клиента                                                              |
| Новый клиент приходит уже во время нагрузки, когда сервер занят множеством других соединений | Новый клиент всё равно получает ответ, потому что главный поток продолжает принимать подключения, а обработка уже принятых клиентов вынесена в отдельные потоки |
| Один отдельный медленный или долгий клиент существует параллельно с остальными               | Такой клиент может тормозить только свой поток, но не блокирует весь сервер и не мешает приёму новых клиентов                                                   |

# 🏗 Архитектура

| 🏗 Компоненты                                                                                                                 | 📖 Роль каждого компонента                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `main`, `acceptor.accept`, `std::thread`, `HandleConnection`, `detach`, `tcp::socket`, `buffer`, `ReadRequest`, `http::write` | `main` управляет жизнью процесса и принимает подключения, `acceptor.accept` получает нового клиента, `std::thread` создаёт отдельный поток, `HandleConnection` обслуживает конкретную клиентскую сессию, `detach` позволяет главному потоку сразу продолжить работу, `tcp::socket` переносит соединение в поток обслуживания, `buffer` хранит данные сессии, `ReadRequest` читает HTTP-запросы, а `http::write` отправляет ответ клиенту |
