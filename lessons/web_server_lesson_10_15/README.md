# 🚀 Практическое занятие №7

## Ошибка Content-Length

---

# 📌 Кратко

| 📌 Что это                                   | 📖 Полное описание                                                                                                                                 |
| -------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| Эксперимент с HTTP заголовком Content-Length | В этом задании сервер намеренно отправляет неправильную длину ответа, из-за чего клиент читает только часть HTML и отображает обрезанный результат |

---

# 🎯 Цель

| 🎯 Что нужно понять              | 📖 Объяснение                                                   |
| -------------------------------- | --------------------------------------------------------------- |
| Content-Length управляет чтением | Клиент читает ровно столько байтов, сколько указано в заголовке |
| Ошибка длины ломает ответ        | Лишние байты игнорируются клиентом                              |
| HTTP работает на уровне байтов   | Браузер не "угадывает", а строго следует заголовкам             |

---

# ⚙️ Основной код

```cpp
response.body() = "<strong>Hello</strong>";
response.content_length(response.body().size() / 2);
response.keep_alive(false);
```

---

# 🧠 Главная идея

| 🧠 Логика                                 | 📖 Смысл                          |
| ----------------------------------------- | --------------------------------- |
| `Content-Length меньше реального размера` | клиент получает только часть HTML |

---

# 🔍 Проверка и результат

| 🔍 Что выполнить                                                                      | 📥 Результат                                                                   |
| ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Открыть `http://127.0.0.1:8080/` в браузере и выполнить `curl http://127.0.0.1:8080/` | Браузер отображает Hel жирным, а curl показывает обрезанный HTML `<strong>Hel` |

---

# 🌐 Что происходит

| 🌐 Этап                             | 📖 Объяснение                     |
| ----------------------------------- | --------------------------------- |
| Сервер отправляет полный HTML       | `<strong>Hello</strong>`          |
| Content-Length уменьшается в 2 раза | клиенту сообщается меньший размер |
| Клиент читает только часть          | `<strong>Hel`                     |
| HTML обрывается                     | закрывающий тег не доходит        |
| Браузер рендерит результат          | отображается Hel жирным           |

---

# 🖥 Поведение curl

| 🖥 Что видно                 | 📖 Почему так                                                        |
| ---------------------------- | -------------------------------------------------------------------- |
| `<strong>Hel` и сразу prompt | curl показывает сырой ответ без форматирования и без перевода строки |
