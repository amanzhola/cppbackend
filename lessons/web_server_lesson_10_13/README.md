# 🚀 Практическое занятие №6

## Обрабатывать только GET

---

# 📌 Кратко

| 📌 Что это                             | 📖 Полное описание                                                                                                                                                    |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Учебный HTTP-сервер с проверкой метода | В этом задании сервер анализирует не только путь запроса, но и HTTP-метод, разрешает только GET и возвращает ошибку 405 Method Not Allowed для всех остальных методов |

---

# 🎯 Цель

| 🎯 Что нужно понять                        | 📖 Что это означает на практике                                                   |
| ------------------------------------------ | --------------------------------------------------------------------------------- |
| Сервер должен учитывать HTTP-метод         | Недостаточно проверять только путь, нужно ещё проверять GET, POST и другие методы |
| Метод влияет на ответ                      | Один и тот же путь может быть разрешён для GET, но запрещён для POST              |
| Неподдерживаемые методы должны отклоняться | Сервер должен возвращать 405 Method Not Allowed и понятное сообщение об ошибке    |

---

# ⚙️ Основной код

```cpp
if (req.method() != http::verb::get) {
    StringResponse response{http::status::method_not_allowed, req.version()};
    response.set(http::field::server, "Lesson10_13");
    response.set(http::field::content_type, "text/plain; charset=utf-8");
    response.keep_alive(false);
    response.body() = "Only GET method is supported\n";
    response.prepare_payload();

    http::write(socket, response, ec);
    socket.shutdown(tcp::socket::shutdown_send, ec);
    return;
}
```

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                              | 📖 Смысл                                                                                             |
| ----------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| `сначала проверяется req.method(), и только потом сервер переходит к маршрутам по req.target()` | Сервер должен сначала убедиться, что метод разрешён, и только после этого анализировать путь запроса |

---

# 🌐 Что именно проверяется

| 🌐 Сценарий  | 📖 Что должен сделать сервер   |
| ------------ | ------------------------------ |
| GET /        | Вернуть HTML Main page         |
| GET /hello   | Вернуть HTML Hello             |
| GET /unknown | Вернуть 404 Not Found          |
| POST /       | Вернуть 405 Method Not Allowed |
| POST /hello  | Вернуть 405 Method Not Allowed |

---

# 🔍 Проверка и результат

| 🔍 Что выполнить                                                                                  | 📥 Какой результат должен быть                                                                                                          |
| ------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить GET запросы к `/` и `/hello`, затем запрос к неизвестному пути и POST запрос к `/hello` | Сервер возвращает HTML для разрешённых GET-маршрутов, 404 для неизвестного пути и 405 с текстом `Only GET method is supported` для POST |

# 🌐 HTTP-смысл задания

| 🌐 Что происходит          | 📖 Почему это важно                                            |
| -------------------------- | -------------------------------------------------------------- |
| GET-запросы обрабатываются | Сервер разрешает поддерживаемый метод                          |
| POST-запросы отклоняются   | Сервер защищает логику приложения от неподдерживаемых действий |
| Статус зависит от метода   | HTTP-метод является частью контракта между клиентом и сервером |

---

# ⚠️ Важно

| ⚠️ Проблема                           | 📖 Причина и решение                                        |
| ------------------------------------- | ----------------------------------------------------------- |
| POST работает как GET                 | Нет проверки метода, нужно проверить `req.method()`         |
| Для POST возвращается неверный статус | Нужно установить `http::status::method_not_allowed`         |
| Клиент не понимает ошибку             | Нужно вернуть понятный текст `Only GET method is supported` |
