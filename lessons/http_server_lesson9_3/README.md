# HTTP Server Lesson 9_3 (accept без HTTP-ответа)

---

## 📌 Описание

В этой версии проекта реализован минимальный TCP-сервер на Boost.Asio, который:

* слушает порт 8080
* ожидает входящее подключение
* принимает соединение через `accept()`
* выводит сообщение о подключении
* завершает работу

⚠️ Важно: сервер **не отправляет HTTP-ответ**, что является частью задания.

---

## 🧠 Цель задания

Понять ключевой момент работы серверов:

👉 разницу между:

* сервер "запущен"
* сервер "принял соединение"

И разобраться, как работает:

```cpp
acceptor.accept(socket);
```

# 2. Код программы

Файл `main.cpp`:

```cpp
#ifdef WIN32
#include <sdkddkver.h>
#endif

#include <boost/asio/ip/tcp.hpp>
#include <iostream>
#include <string_view>

namespace net = boost::asio;
using tcp = net::ip::tcp;
using namespace std::literals;

int main() {
    net::io_context ioc;

    const auto address = net::ip::make_address("0.0.0.0");
    constexpr unsigned short port = 8080;

    tcp::acceptor acceptor(ioc, {address, port});

    std::cout << "Waiting for socket connection"sv << std::endl;

    tcp::socket socket(ioc);
    acceptor.accept(socket);

    std::cout << "Connection received"sv << std::endl;
}
```


---

# 6. Почему возникает ошибка у curl

Это **ожидаемое поведение**.

Сервер:

* принял соединение ✅
* но **не отправил HTTP-ответ ❌**
* завершил программу ❌

В результате:

* соединение закрывается
* `curl` не получает HTTP-ответ
* появляется ошибка

---

## 💡 Важно понять

Ошибка `curl` не означает, что программа сломалась.

Она означает:

👉 сервер не отправил HTTP-ответ

---

# 7. Что делает accept()

```cpp
acceptor.accept(socket);
```

Это ключевая строка.

Она:

* блокирует выполнение программы
* ждёт входящее подключение
* возвращает управление только после подключения клиента

---

# 8. Поведение программы

### До accept():

```text
сервер слушает порт
```

### Во время accept():

```text
сервер ждёт клиента (блокируется)
```

### После подключения:

```text
accept() завершается
сервер получает socket
```

---

# 9. Что показывает это упражнение

Это задание демонстрирует:

1. как сервер слушает порт
2. как происходит блокирующее ожидание
3. как клиент инициирует соединение
4. что происходит после подключения
5. что HTTP — это следующий уровень поверх TCP

# 10. Ограничения сервера

Этот сервер:

* не читает данные от клиента
* не отправляет ответ
* обрабатывает только одно соединение
* завершает работу сразу после подключения
