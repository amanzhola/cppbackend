# 🚀 Практическое занятие №1

## Сервер, который не умирает после первого клиента

---

## 📌 Обзор

| 🚀 Параметр | 📥 Значение       | 🚀 Параметр | 📥 Значение           |
| ----------- | ----------------- | ----------- | --------------------- |
| Тип         | TCP / HTTP Server | Порт        | 8080                  |
| Библиотека  | Boost.Asio        | Адрес       | 0.0.0.0               |
| Режим       | Blocking (sync)   | Особенность | Множественные клиенты |

---

## 🎯 Цель задания

| 🎯 Что изучаем | 📖 Объяснение             | 🎯 Что изучаем | 📖 Объяснение              |
| -------------- | ------------------------- | -------------- | -------------------------- |
| Один сеанс     | Обработка одного клиента  | Жизнь сервера  | Сервер не завершает работу |
| Accept loop    | Постоянный приём клиентов | Blocking I/O   | Ожидание подключения       |

---

## 🧠 Ключевая идея

| 💡 Поведение сервера                                                                                |
| --------------------------------------------------------------------------------------------------- |
| `while (true)<br>tcp::socket socket(ioc);<br>acceptor.accept(socket);<br>HandleConnection(socket);` |

---

## ⚙️ Основной код

```cpp
int main() {
    net::io_context ioc;

    const auto address = net::ip::make_address("0.0.0.0");
    constexpr unsigned short port = 8080;

    tcp::acceptor acceptor(ioc, {address, port});

    while (true) {
        tcp::socket socket(ioc);
        acceptor.accept(socket);
        HandleConnection(socket);
    }
}
```

---

## 🔍 Что происходит

| 🔍 Событие         | ⚙️ Поведение        | 🔍 Событие        | ⚙️ Поведение        |
| ------------------ | ------------------- | ----------------- | ------------------- |
| Клиент подключился | accept() возвращает | Клиент отключился | socket уничтожается |
| Новый клиент       | принимается в цикле | Сервер            | продолжает работать |
