# 🚀 Практическое занятие №1

## Сервер, который не умирает после первого клиента

---

# 📌 Кратко

| 📌 Что это                                                                                | 📌 Описание                                                                                                                                                                                                                                                                                  |
| ----------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Практическая работа по серверу, который не завершается после обслуживания первого клиента | В этой задаче нужно показать, что сервер живёт дольше одного соединения, постоянно принимает новые подключения через цикл `while (true)`, использует `acceptor.accept(socket)` для блокирующего ожидания клиента и после завершения одного сеанса продолжает работать дальше без перезапуска |

---

# 🎯 Цель

| 🎯 Что нужно понять                                                         | 📖 Что это означает на практике                                                                                                                                                                                                             |
| --------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Разницу между одним клиентским сеансом и жизнью всего серверного приложения | Один запрос или один клиент не должен завершать процесс сервера; сервер обязан оставаться запущенным, снова ждать новое подключение, снова вызывать обработку соединения и отвечать на последующие запросы столько раз, сколько потребуется |

---

# ⚙️ Основной код

```cpp
int main() {
    net::io_context ioc;

    const auto address = net::ip::make_address("0.0.0.0");
    constexpr unsigned short port = 8080;

    tcp::acceptor acceptor(ioc, {address, port});

    while (true) {
        tcp::socket socket(ioc);
        acceptor.accept(socket);
        HandleConnection(socket);
    }
}
```

---

# 🧠 Что здесь изучается

| 🧠 Концепция                                   | 📖 Объяснение                                                                                                                                                                                                                                                                              |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Цикл приёма подключений `while (true)`         | Сервер не ограничивается одним клиентом; после завершения обработки текущего соединения создаётся новый `tcp::socket`, выполняется новый `acceptor.accept(socket)`, затем снова вызывается `HandleConnection(socket)`, и приложение продолжает работу без завершения после первого запроса |
| Блокирующая операция `acceptor.accept(socket)` | Эта функция останавливает выполнение в текущем месте и ждёт, пока подключится новый клиент; когда клиент приходит, сокет заполняется подключением, после чего управление возвращается в программу и начинается обработка                                                                   |
| Разделение “жизнь клиента” и “жизнь сервера”   | Клиент может подключиться, получить ответ и отключиться, но это не означает завершение сервера; сервер существует независимо от одного конкретного подключения и продолжает принимать следующих клиентов                                                                                   |
| Вызов `HandleConnection(socket)`               | В этом месте происходит обработка одного конкретного соединения: чтение запроса, формирование ответа и завершение сеанса; после этого сервер не умирает, а возвращается к ожиданию следующего подключения                                                                                  |

# 🏗 Архитектура

| 🏗 Компоненты                                                                 | 📖 Роль каждого компонента                                                                                                                                                                                                                                                                                                           |
| ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `net::io_context`, `tcp::acceptor`, `tcp::socket`, `HandleConnection(socket)` | `net::io_context` обслуживает операции ввода-вывода; `tcp::acceptor` слушает адрес `0.0.0.0:8080` и принимает входящие подключения; `tcp::socket` представляет одно клиентское соединение; `HandleConnection(socket)` обрабатывает конкретный запрос клиента, после чего цикл сервера возвращается к ожиданию следующего подключения |
