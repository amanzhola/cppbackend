# 🚀 Практическое занятие №1

## Сервер, который не умирает после первого клиента

---

## 📌 Обзор

| Параметр     | Значение                        |
| ------------ | ------------------------------- |
| Тип          | TCP / HTTP сервер               |
| Библиотека   | Boost.Asio                      |
| Порт         | 8080                            |
| Адрес        | 0.0.0.0                         |
| Режим работы | Синхронный (blocking)           |
| Особенность  | Обрабатывает множество клиентов |

---

## 🎯 Цель задания

| Что нужно понять | Описание                                     |
| ---------------- | -------------------------------------------- |
| Один сеанс       | Обработка одного клиента                     |
| Жизнь сервера    | Сервер не должен завершаться после клиента   |
| Accept loop      | Сервер постоянно принимает новые подключения |

---

## 🧠 Ключевая идея

Сервер работает в **бесконечном цикле**, принимая подключения:

```cpp
while (true) {
    tcp::socket socket(ioc);
    acceptor.accept(socket);
    HandleConnection(socket);
}
```

👉 Это означает:

| Поведение               | Результат                   |
| ----------------------- | --------------------------- |
| Один клиент подключился | Обработан                   |
| Клиент отключился       | Сервер продолжает работать  |
| Новый клиент            | Принимается без перезапуска |

---

## ⚙️ Основной код

```cpp
int main() {
    net::io_context ioc;

    const auto address = net::ip::make_address("0.0.0.0");
    constexpr unsigned short port = 8080;

    tcp::acceptor acceptor(ioc, {address, port});

    while (true) {
        tcp::socket socket(ioc);
        acceptor.accept(socket);
        HandleConnection(socket);
    }
}
```

---

## 🔍 Что здесь изучается

| Концепция                 | Объяснение                         |
| ------------------------- | ---------------------------------- |
| `acceptor.accept(socket)` | Блокирующий вызов ожидания клиента |
| `tcp::socket`             | Соединение с конкретным клиентом   |
| Бесконечный цикл          | Сервер не завершается              |
| HandleConnection          | Логика обработки запроса           |

## 📊 Результат проверки

| Действие                 | Ожидаемый результат |
| ------------------------ | ------------------- |
| Первый запрос            | Сервер отвечает     |
| Второй запрос            | Сервер отвечает     |
| Третий запрос            | Сервер отвечает     |
| После нескольких вызовов | Сервер НЕ падает    |

---

## 🖥 Поведение сервера

| Событие            | Что происходит                 |
| ------------------ | ------------------------------ |
| Клиент подключился | accept() возвращает управление |
| Обработка запроса  | HandleConnection               |
| Клиент отключился  | socket уничтожается            |
| Новый клиент       | цикл продолжается              |

---

## ⚠️ Важный момент

Если убрать цикл:

```cpp
acceptor.accept(socket);
HandleConnection(socket);
```

👉 Сервер:

* обработает **только одного клиента**
* **завершится после первого запроса ❌**

---

## 🏗 Архитектура

| Компонент        | Роль                      |
| ---------------- | ------------------------- |
| io_context       | Управление вводом/выводом |
| acceptor         | Приём новых подключений   |
| socket           | Соединение с клиентом     |
| HandleConnection | Обработка запроса         |

---

## 🧾 Итог

| Что достигнуто                  | Статус |
| ------------------------------- | ------ |
| Сервер принимает подключения    | ✅      |
| Работает в цикле                | ✅      |
| Не падает после клиента         | ✅      |
| Обрабатывает несколько запросов | ✅      |

---

## 🏁 Вывод

Сервер должен:

* работать **дольше одного клиента**
* иметь **цикл accept**
* отделять:

  * обработку соединения
  * жизненный цикл приложения

👉 Это базовый фундамент любого backend-сервера.
