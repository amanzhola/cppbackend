# 🚀 Полный учебный вариант программы

## Сокет, буфер, запрос и ответ

---

# 📌 Кратко

| 📌 Что это          | 📖 Полное описание                                                                                                                                                                                                                    |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Учебный HTTP-сервер | Программа показывает полный цикл работы сервера: чтение HTTP-запроса из TCP-соединения, разбор его структуры, вывод информации о запросе в консоль, создание HTTP-ответа и отправку результата клиенту с соблюдением правил протокола |

---

# 🎯 Цель

| 🎯 Что нужно понять                                         | 📖 Что это означает на практике                                                                                                                                                                |
| ----------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Разницу между сущностями socket, buffer, request и response | Серверная программа состоит из нескольких уровней: сокет передаёт данные, буфер хранит сырые байты, request представляет уже разобранный HTTP-запрос, а response является готовым HTTP-ответом |
| Архитектуру сервера                                         | Сервер работает по цепочке: чтение запроса, обработка данных, формирование ответа и отправка клиенту                                                                                           |
| Поведение HTTP                                              | Сервер следует правилам протокола и управляет соединением в зависимости от ситуации                                                                                                            |

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                                                                                                                                                                               | 📖 Смысл                                                                                                                                                                                              |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `while (auto request = ReadRequest(socket, buffer)) { DumpRequest(*request); http::write(socket, response); if (response.need_eof()) break; } // полный цикл обработки одного соединения от чтения запроса до возможного завершения HTTP-сессии` | Сервер читает HTTP-запрос, выводит его структуру, формирует ответ и отправляет его клиенту, после чего решает, продолжать работу с соединением или завершить его в соответствии с правилами протокола |

---

# 🌐 Сущности

| 🌐 Сущность        | 📖 Объяснение                                                                     |
| ------------------ | --------------------------------------------------------------------------------- |
| tcp::socket        | Это сетевое соединение между клиентом и сервером, через которое передаются данные |
| beast::flat_buffer | Это временное хранилище байтов, считанных из сокета до их обработки               |
| StringRequest      | Это уже разобранный HTTP-запрос, содержащий метод, путь, заголовки и тело         |
| StringResponse     | Это HTTP-ответ, который сервер формирует и отправляет клиенту                     |

---

# ⚙️ Архитектура

| ⚙️ Компонент     | 📖 Роль в программе                                          |
| ---------------- | ------------------------------------------------------------ |
| ReadRequest      | Читает HTTP-запрос из сокета и возвращает его как объект     |
| DumpRequest      | Выводит метод, путь и заголовки запроса в консоль            |
| HandleConnection | Управляет циклом обработки запросов внутри одного соединения |
| main             | Принимает клиентов и передаёт соединение в обработчик        |

| curl -v -H "X-Test: demo" http://localhost:8080/ | В логах появляется дополнительный заголовок X-Test и подтверждается чтение пользовательских заголовков |

---

# 🌐 HTTP смысл

| 🌐 Что происходит             | 📖 Почему это важно                                    |
| ----------------------------- | ------------------------------------------------------ |
| Клиент отправляет HTTP-запрос | Сервер получает байты через TCP-соединение             |
| Сервер разбирает запрос       | Данные превращаются в структурированный объект request |
| Сервер формирует response     | Ответ создаётся осмысленно, а не как случайная строка  |
| Сервер отправляет ответ       | Клиент получает корректный HTTP-ответ                  |
