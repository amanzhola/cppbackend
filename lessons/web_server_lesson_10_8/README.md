# 🚀 Полный учебный вариант программы

## Сокет, буфер, запрос и ответ

---

# 📌 Кратко

| 📌 Что это          | 📖 Описание                                                                                                                   |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| Учебный HTTP-сервер | Программа показывает полный цикл работы сервера: чтение HTTP-запроса, вывод его структуры, создание ответа и отправка клиенту |

---

# 🎯 Цель

| 🎯 Что понять            | 📖 Объяснение                                         |
| ------------------------ | ----------------------------------------------------- |
| Разницу между сущностями | socket, buffer, request, response — это разные уровни |
| Архитектуру сервера      | чтение → обработка → ответ                            |
| Поведение HTTP           | сервер работает по правилам протокола                 |

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                                                                             | 📖 Смысл                                                                                              |
| ---------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| `while (auto request = ReadRequest(socket, buffer)) { DumpRequest(*request); http::write(socket, response); if (response.need_eof()) break; }` | Сервер читает запрос, печатает его, отправляет ответ и завершает соединение только если это требуется |

---

# 🌐 Сущности

| 🌐 Что      | 📖 Объяснение                        |
| ----------- | ------------------------------------ |
| tcp::socket | соединение между клиентом и сервером |
| flat_buffer | временное хранение байтов            |
| request     | разобранный HTTP-запрос              |
| response    | HTTP-ответ                           |

---

# ⚙️ Архитектура

| ⚙️ Компонент     | 📖 Роль            |
| ---------------- | ------------------ |
| ReadRequest      | читает HTTP-запрос |
| DumpRequest      | выводит запрос     |
| HandleConnection | цикл обработки     |
| main             | принимает клиентов |

---

# 🔍 Проверка

| 🔍 Команда                                         | 📥 Результат            |
| -------------------------------------------------- | ----------------------- |
| `curl -v http://localhost:8080/`                   | сервер отвечает HTML    |
| `curl -v http://localhost:8080/hello`              | тот же ответ            |
| `curl -v -H "X-Test: demo" http://localhost:8080/` | заголовок виден в логах |

---

# 🖥 Логи

| 🖥 Что видно | 📖 Смысл                  |
| ------------ | ------------------------- |
| GET /        | сервер читает метод       |
| Host: ...    | читает заголовки          |
| X-Test: demo | видит кастомные заголовки |

---

# 🌐 HTTP смысл

| 🌐 Что происходит        | 📖 Почему важно              |
| ------------------------ | ---------------------------- |
| клиент отправляет запрос | сервер читает байты          |
| сервер строит response   | ответ формируется осмысленно |
