# 🚀 Практическое занятие №5

## Сервер закрывает соединение по правилам

---

# 📌 Кратко

| 📌 Что это                                           | 📖 Полное описание                                                                                                                                                                                                                                                                                                                                                       |
| ---------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Сервер с корректной логикой закрытия HTTP-соединения | В этом задании сервер не просто читает запрос и отправляет ответ, а учитывает правила keep-alive, копирует поведение соединения из запроса в ответ через `res.keep_alive(req.keep_alive())`, вычисляет необходимость закрытия через `bool close = !res.keep_alive();` и завершает текущую HTTP-сессию, если этого требует протокол, клиент или внутренняя логика сервера |

---

# 🎯 Цель

| 🎯 Что нужно понять                                 | 📖 Что это означает на практике                                                                                                                        |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Не всегда сервер должен держать соединение открытым | После отправки ответа сервер обязан проверить, нужно ли продолжать цикл обработки запросов в рамках текущего TCP-соединения или нужно завершить сессию |
| Управление keep-alive                               | Сервер должен учитывать желание клиента сохранить или закрыть соединение                                                                               |
| Завершение HTTP-сессии                              | Выход из цикла обработки происходит по правилам протокола                                                                                              |

---

# 🧠 Главная идея урока

| 🧠 Ключевая логика                                                                        | 📖 Смысл                                                                                                                                                                                                                                                                 |
| ----------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `res.keep_alive(req.keep_alive()); bool close = !res.keep_alive(); if (close) { break; }` | Сервер сначала переносит правило keep-alive из запроса в ответ, затем вычисляет, нужно ли закрывать соединение после отправки ответа, и если соединение больше не должно оставаться открытым, выходит из внутреннего цикла обработки запросов и завершает текущую сессию |

---

# ⚙️ Основной код логики закрытия соединения

```cpp
auto res = MakeResponse(req);
bool close = !res.keep_alive();

http::write(socket, res);

if (close) {
    break;
}
```

---

# 🌐 Что именно изучается

| 🌐 Концепция                     | 📖 Объяснение                                                              |
| -------------------------------- | -------------------------------------------------------------------------- |
| res.keep_alive(req.keep_alive()) | Ответ наследует правило жизненного цикла соединения из входящего запроса   |
| bool close = !res.keep_alive()   | После подготовки ответа сервер вычисляет необходимость закрытия соединения |
| if (close) { break; }            | При необходимости сервер завершает обработку текущего соединения           |
| Закрытие соединения              | Это полноценная логика сетевого протокола, а не просто отправка строки     |

# 🌐 Поведение сервера

| 🌐 Сценарий             | 📖 Что делает сервер                                 |
| ----------------------- | ---------------------------------------------------- |
| GET /hello              | Сервер отвечает и может оставить соединение открытым |
| GET с Connection: close | Сервер отвечает и закрывает соединение               |
| Ошибка                  | Сервер завершает текущую сессию                      |

# ⚠️ Что будет без этой логики

| ❌ Ошибка                | 💥 Последствие                    |
| ----------------------- | --------------------------------- |
| Нет проверки keep-alive | соединение ведёт себя неправильно |
| Нет break               | сервер не завершает сессию        |
| Игнорирование протокола | некорректное поведение HTTP       |
