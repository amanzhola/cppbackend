# 🚀 Практическое занятие №5

## Сервер закрывает соединение по правилам

---

# 📌 Кратко

| 📌 Что это                                    | 📖 Полное описание                                                                                                                                                                                                                                           |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Сервер с управлением соединением и keep-alive | В этом задании сервер не просто отправляет ответ клиенту, а полностью управляет жизненным циклом HTTP-соединения, копирует поведение keep-alive из запроса, вычисляет необходимость закрытия соединения и завершает сессию строго по правилам протокола HTTP |

---

# 🎯 Цель

| 🎯 Что нужно понять    | 📖 Что это означает на практике                                                      |
| ---------------------- | ------------------------------------------------------------------------------------ |
| Управление соединением | Сервер не обязан держать соединение открытым бесконечно и должен уметь его завершать |
| Keep-alive логика      | Сервер учитывает поведение клиента и HTTP-протокол                                   |
| Завершение сессии      | Сервер завершает соединение через break внутри цикла обработки                       |

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                                                                                                 | 📖 Смысл                                                                                                                                                                                        |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `res.keep_alive(req.keep_alive()); bool close = !res.keep_alive(); if (close) { break; } // решение о завершении соединения принимается после формирования ответа` | Сервер сначала переносит keep-alive из запроса в ответ, затем вычисляет необходимость закрытия соединения и завершает текущую HTTP-сессию, если соединение больше не должно оставаться открытым |

---

# ⚙️ Основной код

```cpp
auto res = MakeResponse(req);
bool close = !res.keep_alive();

http::write(socket, res);

if (close) {
    break;
}
```

---

# 🌐 Что изучается

| 🌐 Концепция                     | 📖 Объяснение                                                   |
| -------------------------------- | --------------------------------------------------------------- |
| res.keep_alive(req.keep_alive()) | Ответ наследует поведение соединения из запроса клиента         |
| bool close = !res.keep_alive()   | Сервер вычисляет необходимость закрытия соединения после ответа |
| if (close) break                 | Завершение обработки соединения                                 |
| HTTP-сессия                      | Соединение живёт не вечно, а по правилам протокола              |

---

# 🌐 Поведение сервера

| 🌐 Сценарий              | 📖 Что делает сервер                                            |
| ------------------------ | --------------------------------------------------------------- |
| GET /hello               | Сервер отвечает и может продолжать работу с этим же соединением |
| Connection: close        | Сервер завершает соединение сразу после ответа                  |
| Ошибка чтения или записи | Сервер завершает текущую HTTP-сессию                            |

---

# 🔍 Проверка

| 🔍 Команда                                                   | 📥 Результат                                              |
| ------------------------------------------------------------ | --------------------------------------------------------- |
| `curl -v http://localhost:8080/hello`                        | Ответ приходит, соединение может оставаться открытым      |
| `curl -v -H "Connection: close" http://localhost:8080/hello` | Ответ приходит, соединение закрывается сразу после ответа |

---

# 🖥 Логи сервера

| 🖥 Что видно                          | 📖 Что это доказывает                      |
| ------------------------------------- | ------------------------------------------ |
| Request: GET /hello, keep_alive=true  | соединение продолжает жить                 |
| Request: GET /hello, keep_alive=false | соединение будет закрыто                   |
| close=true                            | сервер завершает цикл обработки соединения |

# 🌐 HTTP смысл

| 🌐 Что происходит                                  | 📖 Почему это важно                       |

| сервер принимает решение о закрытии                | это часть правильной архитектуры backend  |
