# 🚀 Практическое занятие №4

## Обработка только разрешённых HTTP-методов

---

# 📌 Кратко

| 📌 Что это                     | 📖 Описание                                                                                                                                                                                               |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Сервер с проверкой HTTP-метода | В этом задании сервер проверяет не только путь запроса через `req.target()`, но и HTTP-метод через `req.method()`, разрешая только `GET` и отклоняя все остальные методы с кодом `405 Method Not Allowed` |

---

# 🎯 Цель

| 🎯 Что нужно понять         | 📖 Что это означает                    |
| --------------------------- | -------------------------------------- |
| HTTP — это не только GET    | Разные методы означают разные действия |
| Проверка метода обязательна | Сервер должен фильтровать запросы      |
| Корректные статусы          | 405 для запрещённых методов            |

---

# 🧠 Главная идея

| 🧠 Логика                                                                       | 📖 Смысл                                                          |
| ------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| `cpp<br>if (req.method() != http::verb::get) {<br>&nbsp;&nbsp;return 405;<br>}` | Сервер сначала проверяет метод и только потом анализирует маршрут |

---
