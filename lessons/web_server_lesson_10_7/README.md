# 🚀 Практическое занятие №5

## Сервер закрывает соединение по правилам

---

# 📌 Кратко

| 📌 Что это                                           | 📖 Описание                                                                                                                                                                                                                                                                                                                                  |
| ---------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Сервер с корректной логикой закрытия HTTP-соединения | В этом задании сервер не просто читает запрос и отправляет ответ, а учитывает правила keep-alive, копирует поведение соединения из запроса в ответ через `res.keep_alive(req.keep_alive())`, вычисляет необходимость закрытия через `bool close = !res.keep_alive();` и завершает текущую HTTP-сессию, если того требует протокол или клиент |

---

# 🎯 Цель

| 🎯 Что нужно понять                                            | 📖 Что это означает на практике                                                                                                                                                                                                          |
| -------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Не всегда сервер должен бесконечно держать соединение открытым | После отправки ответа сервер обязан проверить, нужно ли продолжать цикл чтения запросов в текущем TCP-соединении или пора завершать сессию, потому что этого хочет клиент, этого требует протокол, так решил сервер или произошла ошибка |

---

# 🧠 Главная идея урока

| 🧠 Ключевая логика                                                                                                                          | 📖 Смысл                                                                                                                                                                                                                                                                                       |
| ------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `cpp<br>res.keep_alive(req.keep_alive());<br>bool close = !res.keep_alive();<br>...<br>if (close) {<br>&nbsp;&nbsp;&nbsp;&nbsp;break;<br>}` | Сервер сначала переносит правило keep-alive из запроса в ответ, затем вычисляет, нужно ли закрывать соединение после отправки ответа, и если соединение больше не должно оставаться открытым, выходит из внутреннего цикла обработки запросов и завершает текущую сессию по правилам протокола |

---

# ⚙️ Основной код логики закрытия соединения

```cpp
auto res = MakeResponse(req);
bool close = !res.keep_alive();

http::write(socket, res, ec);

if (close) {
    break;
}
```

---

# 🌐 Что именно изучается

| 🌐 Концепция                       | 📖 Объяснение                                                                                                                                           |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `res.keep_alive(req.keep_alive())` | Ответ наследует правило жизненного цикла соединения из входящего запроса, поэтому сервер ведёт себя согласованно с ожиданиями клиента и HTTP-протоколом |
| `bool close = !res.keep_alive()`   | После подготовки ответа сервер явно вычисляет, нужно ли закрыть соединение после отправки текущего ответа                                               |
| `if (close) { break; }`            | Если keep-alive отключён, сервер прекращает внутренний цикл обработки запросов в рамках текущего соединения и переходит к завершению сессии             |
| Закрытие соединения                | Это уже не просто “ответить строкой”, а полноценная логика сетевого протокола, где важно правильно управлять жизненным циклом TCP-соединения            |

---

# 🌐 Поведение сервера

| 🌐 Сценарий                                   | 📥 Что делает сервер                                                                                                        |
| --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| Обычный `GET /hello` с keep-alive             | Сервер отвечает, может оставить соединение открытым и быть готовым к следующему запросу в рамках той же HTTP-сессии         |
| `GET /hello` с заголовком `Connection: close` | Сервер отвечает, выставляет `keep_alive=false`, вычисляет `close=true` и завершает текущее соединение после отправки ответа |
| Ошибка чтения или записи                      | Сервер завершает текущую сессию, потому что продолжать обработку соединения уже нельзя                                      |

# ⚠️ Что будет, если логику close не сделать

| ❌ Неправильный вариант                                                                           | 💥 Что сломается                                                                                                                                           |
| ------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Если после отправки ответа не проверять `res.keep_alive()` и не завершать цикл при необходимости | Сервер может игнорировать правила протокола, держать соединение открытым тогда, когда его нужно закрыть, и работать некорректно с точки зрения HTTP-сеанса |
