# 🚀 Практическое занятие №5

## Сервер закрывает соединение по правилам

---

# 📌 Кратко

| 📌 Что это                                    | 📖 Полное описание                                                                                                                                                                                                                                                                    |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Сервер с управлением соединением и keep-alive | В этом задании сервер не просто отправляет HTTP-ответ клиенту, а полностью управляет жизненным циклом соединения, копирует поведение keep-alive из запроса, вычисляет необходимость закрытия соединения и завершает HTTP-сессию строго по правилам протокола HTTP и поведения клиента |

---

# 🎯 Цель

| 🎯 Что нужно понять    | 📖 Что это означает на практике                                                                                       |
| ---------------------- | --------------------------------------------------------------------------------------------------------------------- |
| Управление соединением | Сервер не должен держать соединение открытым бесконечно и обязан уметь его корректно завершать после обработки ответа |
| Keep-alive логика      | Сервер принимает решение о закрытии соединения на основе запроса клиента и внутренней логики                          |
| Завершение HTTP-сессии | Выход из цикла обработки происходит строго по правилам протокола HTTP                                                 |

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                                                                                                            | 📖 Смысл                                                                                                                                                                                                                                     |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `res.keep_alive(req.keep_alive()); bool close = !res.keep_alive(); if (close) { break; } // решение принимается после отправки ответа клиенту и анализа состояния соединения` | Сервер сначала копирует поведение keep-alive из запроса клиента в ответ, затем вычисляет необходимость закрытия соединения и завершает текущую HTTP-сессию, если соединение больше не должно оставаться открытым согласно правилам протокола |

---

# ⚙️ Основной код

```cpp
auto res = MakeResponse(req);
bool close = !res.keep_alive();

http::write(socket, res);

if (close) {
    break;
}
```

---

# 🌐 Что изучается

| 🌐 Концепция                     | 📖 Объяснение                                                                                               |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| res.keep_alive(req.keep_alive()) | Ответ наследует поведение соединения из входящего HTTP-запроса клиента и работает согласованно с протоколом |
| bool close = !res.keep_alive()   | Сервер вычисляет необходимость закрытия соединения после отправки HTTP-ответа                               |
| if (close) break                 | Завершение текущей HTTP-сессии через выход из цикла обработки соединения                                    |
| Управление соединением           | Это полноценная логика сетевого протокола, а не просто отправка текста клиенту                              |

---

# 🌐 Поведение сервера

| 🌐 Сценарий              | 📖 Что делает сервер                                                                              |
| ------------------------ | ------------------------------------------------------------------------------------------------- |
| GET /hello               | Сервер отправляет ответ и может продолжить работу с этим же соединением, если keep-alive разрешён |
| GET с Connection: close  | Сервер отправляет ответ и завершает соединение сразу после ответа                                 |
| Ошибка чтения или записи | Сервер завершает текущую HTTP-сессию и закрывает соединение                                       |

# 🔍 Проверка

| 🔍 Команда                                                   | 📥 Результат                                                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------- |
| `curl -v http://localhost:8080/hello`                        | Сервер возвращает ответ Hello и соединение может оставаться открытым в рамках текущей HTTP-сессии |
| `curl -v -H "Connection: close" http://localhost:8080/hello` | Сервер возвращает ответ и сразу закрывает соединение после отправки ответа                        |

---

# 🖥 Логи сервера

| 🖥 Что видно                          | 📖 Что это доказывает                                                |
| ------------------------------------- | -------------------------------------------------------------------- |
| Request: GET /hello, keep_alive=true  | соединение остаётся открытым и сервер готов принять следующий запрос |
| Request: GET /hello, keep_alive=false | соединение будет закрыто после отправки ответа                       |
| close=true                            | сервер завершает цикл обработки соединения и закрывает HTTP-сессию   |

---

# 🌐 HTTP смысл

| 🌐 Что происходит                                             | 📖 Почему это важно                                            |
| ------------------------------------------------------------- | -------------------------------------------------------------- |
| клиент может управлять соединением через заголовок Connection | сервер обязан учитывать это и корректно управлять соединением  |
| сервер принимает решение о закрытии                           | это часть правильной архитектуры HTTP-сервера и backend логики |
