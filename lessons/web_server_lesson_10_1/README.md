# 🌐 HTTP Server Lesson 10_1

## Echo JSON API

---

# 🚀 Обзор

| 📌 Параметр | 💡 Полное описание                                                               |
| ----------- | -------------------------------------------------------------------------------- |
| Тип сервера | HTTP сервер на Boost.Beast, который принимает входящие запросы и обрабатывает их |
| Формат      | Сервер работает как JSON API и возвращает ответы в формате JSON                  |
| Протокол    | Используется TCP соединение для передачи данных                                  |
| Порт        | Сервер слушает порт 8080 и принимает подключения от клиентов                     |
| Метод       | Основной сценарий работы построен вокруг POST запроса                            |

---

# 🧠 Возможности

| 📥 Входной запрос          | 🔁 Что делает сервер                                 | 📤 Результат                        |
| -------------------------- | ---------------------------------------------------- | ----------------------------------- |
| POST /echo с телом запроса | Сервер читает body запроса и извлекает строку        | Возвращает JSON с тем же содержимым |
| Любой другой запрос        | Сервер не выполняет обработку и переходит в fallback | Возвращает JSON с ошибкой           |

---

# 🌐 Endpoint

| 🌐 Запрос                   | 📥 Что возвращает сервер                                                           |
| --------------------------- | ---------------------------------------------------------------------------------- |
| POST /echo                  | Сервер возвращает JSON вида {"echo":"..."} и подставляет туда тело запроса клиента |
| Любой другой путь или метод | Сервер возвращает JSON {"error":"Use POST /echo"} как сообщение об ошибке          |

---

# 🔍 Проверка и результат

| 🔍 Что выполнить                                               | 📥 Какой результат должен быть                                                                                   |
| -------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| curl -X POST http://localhost:8080/echo -d "Hello from client" | Сервер возвращает JSON {"echo":"Hello from client"} и тем самым подтверждает, что тело запроса успешно прочитано |

---

# 🖥 Логи сервера

| 🖥 Что видно в терминале сервера                         | 📖 Что это означает                                                                                              |
| -------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| Connection received, POST /echo, Body: Hello from client | Сервер принял соединение, распознал POST запрос, прочитал тело запроса и использовал его для формирования ответа |

---

# 🌐 HTTP ответ

| 🌐 Что получает клиент                                                           | 📖 Почему это важно                                                                                      |
| -------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| HTTP 200 OK с Content-Type application/json и телом {"echo":"Hello from client"} | Это показывает, что сервер правильно формирует HTTP-ответ и возвращает JSON в соответствии с API логикой |

---

# 🧩 Логика обработки

| 🧩 Шаг обработки запроса | 📖 Что происходит                            |
| ------------------------ | -------------------------------------------- |
| Accept соединение        | Сервер принимает входящее TCP-соединение     |
| Read HTTP request        | Сервер читает HTTP-запрос и извлекает данные |
| Проверка метода          | Сервер проверяет, что используется POST      |
| Проверка пути            | Сервер проверяет, что путь равен /echo       |
| Чтение body              | Сервер извлекает req.body()                  |
| Формирование ответа      | Сервер формирует JSON с тем же содержимым    |
| Отправка ответа          | Сервер отправляет HTTP-ответ клиенту         |

# 🧪 Примеры

| 🧪 Команда                                       | 📥 Результат                                      |
| ------------------------------------------------ | ------------------------------------------------- |
| curl -X POST http://localhost:8080/echo -d "abc" | Сервер возвращает JSON {"echo":"abc"}             |

---

# ⚠️ Важно

| ⚠️ Ошибка               | 📖 Причина и решение                                     |
| ----------------------- | -------------------------------------------------------- |
| 404 или ошибка маршрута | Используется неверный путь, нужно использовать /echo     |
| Пустой ответ            | Не передано тело запроса, нужно использовать параметр -d |
| Метод не POST           | Используется GET вместо POST, нужно явно указать -X POST |
