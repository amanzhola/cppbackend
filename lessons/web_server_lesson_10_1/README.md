# 🌐 HTTP Server Lesson 10_1 (Echo JSON API)

---

## 🚀 Обзор

| 📌 Параметр | 💡 Значение        |
| ----------- | ------------------ |
| Тип сервера | HTTP (Boost.Beast) |
| Формат      | JSON API           |
| Протокол    | TCP                |
| Порт        | 8080               |
| Метод       | POST               |

---

## 🧠 Возможности

| 📥 Input            | 🔁 Обработка | 📤 Output       |
| ------------------- | ------------ | --------------- |
| POST /echo          | читает body  | возвращает JSON |
| любой другой запрос | fallback     | ошибка          |

---

# 🌐 Endpoint

| Метод | URL          | Ответ                        |
| ----- | ------------ | ---------------------------- |
| POST  | `/echo`      | `{"echo":"..."}`             |
| ANY   | любой другой | `{"error":"Use POST /echo"}` |

---

# 🔍 Проверка и результат

| 🔍 Запрос                                                                | 📥 Ответ                               |
| ------------------------------------------------------------------------ | -------------------------------------- |
| `bash<br>curl -X POST http://localhost:8080/echo -d "Hello from client"` | `json<br>{"echo":"Hello from client"}` |

---

# 🖥 Логи и HTTP

| 🖥 Логи сервера                                                        | 🌐 HTTP Ответ                                                                                   |
| ---------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `text<br>Connection received<br>POST /echo<br>Body: Hello from client` | `http<br>HTTP/1.1 200 OK<br>Content-Type: application/json<br><br>{"echo":"Hello from client"}` |

---

# 🧩 Логика обработки

| Шаг | Действие                 |
| --- | ------------------------ |
| 1   | accept соединение        |
| 2   | read HTTP request        |
| 3   | проверить method = POST  |
| 4   | проверить target = /echo |
| 5   | взять req.body()         |
| 6   | вернуть JSON             |

---

# ⚠️ Важно

| Ошибка        | Причина        | Решение                |
| ------------- | -------------- | ---------------------- |
| 404           | неверный путь  | использовать `/echo`   |
| пустой ответ  | нет body       | передать `-d`          |
| метод не POST | неверный метод | использовать `-X POST` |
