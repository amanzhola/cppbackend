# 🚀 Практическое занятие №2

## Научиться видеть запросы клиента

---

# 📌 Кратко

| 📌 Что это                                       | 📖 Полное описание                                                                                                                                                                                                           |
| ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Учебный HTTP-сервер для анализа запросов клиента | В этом задании сервер не просто отвечает клиенту, а читает входящий HTTP-запрос, выводит в консоль его метод, путь и заголовки, после чего формирует ответ и тем самым показывает реальную внутреннюю структуру HTTP-запроса |

---

# 🎯 Цель

| 🎯 Что нужно понять         | 📖 Что это означает на практике                                                                          |
| --------------------------- | -------------------------------------------------------------------------------------------------------- |
| HTTP-запрос имеет структуру | Запрос состоит не просто из факта подключения, а из метода, пути и набора заголовков                     |
| Сервер может видеть запрос  | Программа может распечатать содержимое HTTP-запроса в консоли сервера                                    |
| Заголовки имеют значение    | Host, User-Agent, Accept и пользовательские заголовки являются частью запроса и видны на стороне сервера |

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                                                                                                                                                                       | 📖 Смысл                                                                                                                                                           |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `while (auto request = ReadRequest(socket, buffer)) { DumpRequest(*request); http::write(socket, response); if (response.need_eof()) break; } // сервер читает запрос, показывает его структуру и только потом отправляет ответ клиенту` | Сервер получает не абстрактное соединение, а полноценный HTTP-запрос, из которого можно извлечь метод, путь и заголовки, после чего уже построить корректный ответ |

---

# 🌐 Что именно видно в запросе

| 🌐 Элемент запроса | 📖 Что показывает сервер                                                                      |
| ------------------ | --------------------------------------------------------------------------------------------- |
| Метод              | Сервер видит, используется ли GET, POST или другой HTTP-метод                                 |
| Путь               | Сервер видит, какой именно URL был запрошен, например `/` или `/hello`                        |
| Заголовки          | Сервер видит Host, User-Agent, Accept и любые дополнительные заголовки клиента                |
| Полная структура   | Вместо ощущения «кто-то пришёл» становится видно, что запрос состоит из конкретных HTTP-полей |

---

# ⚙️ Основной код

```cpp id="mk10917"
void DumpRequest(const StringRequest& req) {
    std::cout << req.method_string() << ' ' << req.target() << std::endl;
    for (const auto& header : req) {
        std::cout << "  "sv << header.name_string() << ": "sv << header.value() << std::endl;
    }
}
```

# 🔍 Проверка и результат

| 🔍 Что выполнить                                                                                                 | 📥 Какой результат должен быть                                                                                                                                      |
| ---------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `curl http://localhost:8080/ ; curl http://localhost:8080/hello ; curl -H "X-Test: demo" http://localhost:8080/` | Сервер выводит в консоль строки вида `GET /`, `GET /hello`, а также показывает стандартные и пользовательские заголовки, например Host, User-Agent, Accept и X-Test |

# 🌐 HTTP-смысл задания

| 🌐 Что происходит             | 📖 Почему это важно                                                     |
| ----------------------------- | ----------------------------------------------------------------------- |
| Клиент отправляет HTTP-запрос | Сервер получает не только соединение, но и осмысленные данные протокола |
| Сервер печатает запрос        | Можно увидеть, как реально устроен HTTP-запрос                          |
| Сервер отвечает клиенту       | После анализа запроса приложение остаётся полноценным HTTP-сервером     |

# ⚠️ Что будет без DumpRequest

| ❌ Неправильный вариант                                      | 💥 Что потеряется                                                                                                      |
| ----------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Сервер читает запрос, но не выводит его структуру в консоль | Тогда не видно метод, путь и заголовки, и учебная цель задания про наблюдение за реальным HTTP-запросом не достигается |

# 🏗 Архитектура
