# 🚀 Практическое занятие №8

## Многопоточный HTTP сервер без блокировок

---

# 📌 Кратко

| 📌 Что это                                            | 📖 Полное описание                                                                                                                                                                                                                                                |
| ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| HTTP сервер с обработкой клиентов в отдельных потоках | В этом задании сервер после каждого `accept` создаёт новый поток, передаёт туда клиентский сокет и сразу возвращается к приёму следующих подключений, поэтому один медленный клиент больше не блокирует остальных и главный поток не перестаёт выполнять `accept` |

---

# 🎯 Цель

| 🎯 Что нужно понять                                     | 📖 Что это означает на практике                                                                                                         |
| ------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| Один клиент не должен блокировать всех остальных        | Если один клиент обслуживается долго, это должно тормозить только его собственный поток, а не весь сервер                               |
| Главный поток должен продолжать принимать клиентов      | После `accept` сервер не должен застревать в обработке одного соединения, а должен сразу возвращаться к ожиданию следующего подключения |
| Обработка клиента может быть вынесена в отдельный поток | Каждый клиент получает собственный контекст выполнения, а сервер становится отзывчивее при нескольких одновременных подключениях        |

---

# ⚙️ Основной код

```cpp
std::thread(
    [](tcp::socket socket) {
        HandleConnection(std::move(socket));
    },
    std::move(socket)
).detach();
```

---

# 🧠 Главная идея

| 🧠 Ключевая логика                                                                       | 📖 Смысл                                                                                                                                                                             |
| ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `accept → создать поток → передать сокет → detach → сразу вернуться к следующему accept` | Сервер больше не связывает приём новых клиентов с завершением обработки старого клиента, поэтому исчезает блокировка приёма новых подключений из-за обслуживания уже открытой сессии |

---

# 🌐 Что именно изменилось после переделки сервера

| 🌐 До изменения                                                                                                           | 📖 После изменения                                                                                                                                                  |
| ------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Один медленный клиент мог удерживать сервер внутри обработки соединения и тем самым тормозить приём следующих подключений | Медленный клиент тормозит только свой поток, а главный поток продолжает выполнять `accept` и принимает новых клиентов независимо от состояния предыдущих соединений |

---

# 🔍 Проверка и результат

| 🔍 Что выполнить                                                                                                                      | 📥 Какой результат должен быть                                                                                                                                           |
| ------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Выполнить несколько запросов подряд и затем несколько запросов одновременно через фоновые `curl` команды или из нескольких терминалов | Сервер отвечает на все запросы, не зависает, не перестаёт принимать подключения и продолжает работать даже когда несколько клиентов обращаются к нему почти одновременно |

---

# 🖥 Что видно в логах сервера

| 🖥 Что видно в терминале сервера                                                                                                                | 📖 Что это доказывает                                                                                                                   |
| ----------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| В логах появляются несколько запросов от разных клиентов, а сервер не прекращает принимать новые соединения после первого обработанного запроса | Главный поток не блокируется обработкой старого клиента, а каждый клиент обслуживается независимо внутри собственного потока выполнения |

---

# 🌐 HTTP и потоковая модель

| 🌐 Что происходит                                                                                   | 📖 Почему это важно                                                                                                                                        |
| --------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Клиент подключается к серверу, сервер принимает сокет и немедленно передаёт его в отдельный поток   | Это позволяет разделить уровень приёма подключений и уровень обработки клиентских сессий, что делает сервер ближе к реальной многоклиентской модели работы |
| Один клиент отправляет запросы внутри своей сессии, а другие клиенты могут подключаться параллельно | Сервер больше не ведёт себя как одноклиентская программа, а начинает обслуживать несколько соединений одновременно                                         |

| ⚠️ Что было плохо раньше                                                                                 | 📖 Что исчезает теперь                                                                                                        |
| -------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| При обработке одного медленного клиента сервер мог надолго застрять и перестать принимать новых клиентов | Исчезает блокировка приёма новых подключений из-за обработки старого клиента, потому что обработка вынесена в отдельный поток |

# 🏗 Архитектура

| 🏗 Компоненты                                                                         | 📖 Роль каждого компонента                                                                                                                                                                                                                                                                                                                |
| ------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `main`, `acceptor.accept`, `std::thread`, `HandleConnection`, `detach`, `tcp::socket` | `main` управляет жизнью процесса и принимает подключения, `acceptor.accept` получает нового клиента, `std::thread` создаёт отдельный поток, `HandleConnection` обслуживает конкретную клиентскую сессию, `detach` позволяет главному потоку сразу продолжить работу, а `tcp::socket` переносит клиентское соединение в поток обслуживания |

| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| После переноса обработки клиента в отдельный поток сервер перестаёт блокировать приём новых подключений из-за старого клиента, а значит становится реально многоклиентским по поведению | Это важный шаг к пониманию практического backend-сервера: главный поток отвечает за приём клиентов, а обработка конкретных соединений может выполняться параллельно и независимо |
