# 🌐 HTTP Server Lesson 10_0_1

## JSON API

---

# 🚀 Обзор

| 📌 Параметр   | 💡 Полное описание                                                        |
| ------------- | ------------------------------------------------------------------------- |
| Тип сервера   | HTTP сервер на Boost.Beast, который принимает запросы и возвращает ответы |
| Формат ответа | Все ответы возвращаются в формате JSON                                    |
| Протокол      | Используется TCP соединение для передачи HTTP-запросов и ответов          |
| Порт          | Сервер работает на порту 8080 и принимает подключения                     |
| Версия        | lesson_10_0_1 показывает переход от текста к JSON API                     |

---

# 🧠 Возможности

| 🌐 Возможность      | 📖 Как это работает                                           |
| ------------------- | ------------------------------------------------------------- |
| Маршрут /           | Возвращает JSON с сообщением Main page                        |
| Маршрут /hello      | Возвращает JSON с сообщением Hello                            |
| Маршрут /bye        | Возвращает JSON с сообщением Goodbye                          |
| Неизвестный маршрут | Возвращает JSON с ошибкой 404                                 |
| API стиль           | Сервер возвращает структурированные JSON ответы вместо текста |

---

# 🌐 Маршруты

| 🌐 Запрос    | 📥 Ответ сервера                               |
| ------------ | ---------------------------------------------- |
| GET /        | Сервер возвращает JSON {"message":"Main page"} |
| GET /hello   | Сервер возвращает JSON {"message":"Hello!"}    |
| GET /bye     | Сервер возвращает JSON {"message":"Goodbye!"}  |
| GET /unknown | Сервер возвращает JSON {"error":"404"}         |

---

# 🔍 Проверка и результат

| 🔍 Что выполнить                                                                                                                     | 📥 Какой результат должен быть                                                                                       |
| ------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| curl http://localhost:8080/ ; curl http://localhost:8080/hello ; curl http://localhost:8080/bye ; curl http://localhost:8080/unknown | Сервер возвращает JSON ответы Main page, Hello, Goodbye и JSON ошибку 404, что подтверждает корректную маршрутизацию |

---

# 🖥 Логи сервера

| 🖥 Что видно в терминале сервера                               | 📖 Что это доказывает                                                                               |
| -------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| Connection received, GET /, GET /hello, GET /bye, GET /unknown | Сервер принимает соединения, читает HTTP-запросы, различает маршруты и формирует разные JSON ответы |

---

# 🌐 HTTP ответ

| 🌐 Что получает клиент                                          | 📖 Почему это важно                                                                        |
| --------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| HTTP 200 OK с Content-Type application/json и JSON телом ответа | Сервер корректно формирует HTTP-ответ и возвращает структурированные данные в формате JSON |

---

# 🧩 Логика обработки

| 🧩 Шаг обработки  | 📖 Что происходит                        |
| ----------------- | ---------------------------------------- |
| Accept соединение | Сервер принимает входящее TCP-соединение |
| Read HTTP request | Сервер читает HTTP-запрос                |
| Анализ req.target | Сервер определяет маршрут запроса        |
| Выбор маршрута    | Сервер выбирает нужную ветку обработки   |
| Формирование JSON | Сервер формирует JSON-ответ              |
| Write response    | Сервер отправляет ответ клиенту          |

---

# 🔥 Сравнение версий

| 🔥 Версия     | 📖 Разница                                |
| ------------- | ----------------------------------------- |
| lesson_10_0   | Сервер возвращает текстовые ответы        |
| lesson_10_0_1 | Сервер возвращает JSON и работает как API |

# ⚠️ Важно

| ⚠️ Проблема              | 📖 Причина и решение                            |
| ------------------------ | ----------------------------------------------- |
| curl зависает            | нет перевода строки в ответе, нужно добавить \n |
| ошибка ответа            | не вызывается http::write                       |
| 404 работает неправильно | не реализован маршрут unknown                   |
