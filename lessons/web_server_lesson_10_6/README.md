# 🚀 Практическое занятие №4

## Обработка только разрешённых HTTP-методов

---

# 📌 Кратко

| 📌 Что это                             | 📖 Полное описание                                                                                                                                                                                                                                                                                 |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Сервер с жёсткой проверкой HTTP-метода | В этом задании сервер анализирует не только путь запроса через `req.target()`, но и HTTP-метод через `req.method()`, разрешает только `GET`, отклоняет `POST`, `PUT` и другие методы с кодом `405 Method Not Allowed` и тем самым реализует базовую защиту API и корректное поведение HTTP-сервера |

---

# 🎯 Цель

| 🎯 Что нужно понять         | 📖 Что это означает на практике                                                                                            |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| HTTP — это не только GET    | Клиент может отправлять разные методы, например `GET`, `POST` и `PUT`, и сервер обязан учитывать это при обработке запроса |
| Проверка метода обязательна | Перед обработкой маршрута сервер должен сначала проверить метод запроса и только потом переходить к анализу пути           |
| Статус 405                  | Неподдерживаемые методы должны явно отклоняться с корректным статусом `405 Method Not Allowed`                             |

---

# 🧠 Главная идея

| 🧠 Логика сервера                                                                                                                                                                                                                       | 📖 Смысл                                                                                                                                                                                          |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `if (req.method() != http::verb::get) { res.result(http::status::method_not_allowed); res.body() = "Only GET method is supported"; } // проверка метода выполняется до маршрутизации и блокирует все неподдерживаемые действия клиента` | Сервер сначала проверяет HTTP-метод, и если он не `GET`, сразу возвращает ошибку `405`, не переходя к маршрутизации, тем самым гарантируя корректное поведение API и правильную обработку запроса |

---

# 🌐 Поведение сервера

| 🌐 Сценарий запроса | 📖 Что делает сервер                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `GET /hello`        | Сервер возвращает `200 OK` и текст `Hello!`, потому что метод разрешён и маршрут существует                      |
| `POST /hello`       | Сервер возвращает `405 Method Not Allowed` и сообщение `Only GET method is supported`, потому что метод запрещён |
| `PUT /time`         | Сервер возвращает `405 Method Not Allowed`, потому что в этой работе поддерживается только метод `GET`           |
| `GET /unknown`      | Сервер возвращает `404 Not Found`, потому что такой маршрут не существует                                        |

# 🔍 Проверка и результат

| 🔍 Проверка (curl)                                                                                                                                          | 📥 Результат                                                                                                                                                             |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `curl http://localhost:8080/hello ; curl -X POST http://localhost:8080/hello ; curl -X PUT http://localhost:8080/time ; curl http://localhost:8080/unknown` | Сервер возвращает `Hello!` для разрешённого GET-запроса, возвращает `Only GET method is supported` для POST и PUT и возвращает `404 Not Found` для неизвестного маршрута |

# 🌐 HTTP-смысл

| 🌐 Что получает клиент                                                                                                                                            | 📖 Почему это важно                                                                                                                                    |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Для разрешённых методов клиент получает `200 OK`, для запрещённых методов получает `405 Method Not Allowed`, а для неизвестного маршрута получает `404 Not Found` | Это соответствует стандарту HTTP: сервер обязан учитывать и метод, и путь, чтобы корректно обрабатывать запросы и честно сообщать клиенту о результате |

---

# ⚠️ Что будет без проверки метода

| ❌ Неправильное поведение     | 💥 Что сломается                                                                                        |
| ---------------------------- | ------------------------------------------------------------------------------------------------------- |
| Сервер принимает все методы  | API становится некорректным и небезопасным, потому что клиент может выполнять неподдерживаемые действия |
| POST возвращает `200 OK`     | Нарушается логика HTTP и клиент получает неверный сигнал о допустимости операции                        |
| Нет `405 Method Not Allowed` | Клиент не понимает, что операция запрещена сервером и почему запрос не должен выполняться               |
