# 🚀 Практическое занятие №4

## Обработка только разрешённых HTTP-методов

---

# 📌 Кратко

| 📌 Что это                             | 📖 Полное описание                                                                                                                                                                                                                                                                                 |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Сервер с жёсткой проверкой HTTP-метода | В этом задании сервер анализирует не только путь запроса через `req.target()`, но и HTTP-метод через `req.method()`, разрешает только `GET`, отклоняет `POST`, `PUT` и другие методы с кодом `405 Method Not Allowed` и тем самым реализует базовую защиту API и корректное поведение HTTP-сервера |

---

# 🎯 Цель

| 🎯 Что нужно понять         | 📖 Что это означает на практике                                                             |
| --------------------------- | ------------------------------------------------------------------------------------------- |
| HTTP — это не только GET    | Клиент может отправлять разные методы (`GET`, `POST`, `PUT`), и сервер должен это учитывать |
| Проверка метода обязательна | Перед обработкой маршрута сервер обязан проверить метод запроса                             |
| Статус 405                  | Неподдерживаемые методы должны явно отклоняться                                             |

---

# 🧠 Главная идея

| 🧠 Логика сервера                                                                                                                                                             | 📖 Смысл                                                                                                                                                           |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `cpp<br>if (req.method() != http::verb::get) {<br>&nbsp;&nbsp;res.result(http::status::method_not_allowed);<br>&nbsp;&nbsp;res.body() = "Only GET method is supported";<br>}` | Сервер сначала проверяет HTTP-метод, и если он не `GET`, сразу возвращает ошибку `405`, не переходя к маршрутизации, тем самым гарантируя корректное поведение API |

---

# 🌐 Поведение сервера

| 🌐 Сценарий запроса | 📥 Что делает сервер                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `GET /hello`        | Сервер возвращает `200 OK` и текст `Hello!`, потому что метод разрешён и маршрут существует                      |
| `POST /hello`       | Сервер возвращает `405 Method Not Allowed` и сообщение `Only GET method is supported`, потому что метод запрещён |
| `PUT /time`         | Сервер возвращает `405 Method Not Allowed`, потому что поддерживается только `GET`                               |
| `GET /unknown`      | Сервер возвращает `404 Not Found`, потому что маршрут не существует                                              |

# 🌐 HTTP-смысл

| 🌐 Что получает клиент                                                                                                                      | 📖 Почему это важно                                                                                             |
| ------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| Для разрешённых методов клиент получает `200 OK`, для запрещённых — `405 Method Not Allowed`, а для неизвестного маршрута — `404 Not Found` | Это соответствует стандарту HTTP: сервер обязан учитывать и метод, и путь, чтобы корректно обрабатывать запросы |

# ⚠️ Что будет без проверки метода

| ❌ Неправильное поведение    | 💥 Что сломается                           |
| --------------------------- | ------------------------------------------ |
| Сервер принимает все методы | API становится некорректным и небезопасным |
| POST возвращает 200         | нарушается логика HTTP                     |
| Нет 405                     | клиент не понимает, что операция запрещена |
