# 🌐 HTTP Server Lesson 10_2 (JSON + HTML)

---

# 🚀 Обзор

| 📌 Параметр | 💡 Значение                                                                                             |
| ----------- | ------------------------------------------------------------------------------------------------------- |
| Тип сервера | HTTP сервер на Boost.Beast, который обрабатывает HTTP-запросы и возвращает ответы в формате JSON и HTML |
| Форматы     | JSON и HTML используются для ответа клиенту в зависимости от маршрута                                   |
| Протокол    | TCP используется как транспортный уровень                                                               |
| Порт        | Сервер работает на порту 8080 и принимает входящие соединения                                           |
| Методы      | Поддерживаются GET и POST запросы для разных маршрутов                                                  |

---

# 🧠 Возможности

| 🌐 Маршрут | 📖 Поведение                             |
| ---------- | ---------------------------------------- |
| /          | возвращает JSON с сообщением Main page   |
| /hello     | возвращает JSON с сообщением Hello       |
| /bye       | возвращает JSON с сообщением Goodbye     |
| /echo      | принимает POST и возвращает тело запроса |
| /html      | возвращает HTML страницу                 |
| unknown    | возвращает JSON с ошибкой Not found      |

---

# 🌐 Маршруты

| Метод | URL     | Ответ                                           | Тип  |
| ----- | ------- | ----------------------------------------------- | ---- |
| GET   | /       | {"message":"Main page"}                         | JSON |
| GET   | /hello  | {"message":"Hello!"}                            | JSON |
| GET   | /bye    | {"message":"Goodbye!"}                          | JSON |
| POST  | /echo   | {"echo":"..."}                                  | JSON |
| GET   | /html   | HTML страница с заголовком Hello from HTML page | HTML |
| ANY   | unknown | {"error":"Not found"}                           | JSON |

---

# 🔍 Проверка и результат

| 🔍 Команда                                                                                                                           | 📥 Результат                                                               |
| ------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------- |
| curl http://localhost:8080/ ; curl http://localhost:8080/hello ; curl http://localhost:8080/bye ; curl http://localhost:8080/unknown | сервер возвращает JSON ответы Main page, Hello, Goodbye и ошибку Not found |

---

# 🔁 Echo и HTML

| 🔍 Команда                                                     | 📥 Результат                                                   |
| -------------------------------------------------------------- | -------------------------------------------------------------- |
| curl -X POST http://localhost:8080/echo -d "Hello from client" | сервер возвращает JSON {"echo":"Hello from client"}            |
| curl http://localhost:8080/html                                | сервер возвращает HTML страницу с текстом Hello from HTML page |

---

# 🖥 Логи сервера

| 🖥 Что видно                                                               | 📖 Что это доказывает                                                                           |
| -------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| Connection received, GET /, POST /echo, Body: Hello from client, GET /html | сервер принимает соединения, различает методы и маршруты, читает body и формирует разные ответы |

# 🌐 HTTP ответ

# 🧩 Логика обработки

| Шаг | Действие                    |
| --- | --------------------------- |
| 1   | accept соединение           |
| 2   | read HTTP request           |
| 3   | проверка method             |
| 4   | проверка target             |
| 5   | выбор формата JSON или HTML |
| 6   | write response              |

# ⚠️ Важно

| Проблема                    | Причина                      | Решение                       |
| --------------------------- | ---------------------------- | ----------------------------- |
| JSON не читается            | неверный Content-Type        | использовать application/json |
| HTML отображается как текст | неверный Content-Type        | использовать text/html        |
| echo не работает            | используется GET вместо POST | использовать POST запрос      |

---

# 🧠 Архитектура

| Поток данных                                                          | Объяснение                                                                                               |
| --------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| Client → TCP socket → HTTP request → routing → JSON или HTML response | сервер принимает запрос, анализирует метод и путь, выбирает формат ответа и отправляет результат клиенту |
