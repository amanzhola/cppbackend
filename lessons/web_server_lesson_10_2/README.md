# 🌐 HTTP Server Lesson 10_2

## JSON + HTML

---

# 🚀 Обзор

| 📌 Что это                                   | 💡 Полное описание                                                                                                                                                                                                                                                                   |
| -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Учебный HTTP-сервер с поддержкой JSON и HTML | В этом задании сервер принимает HTTP-запросы, анализирует метод и путь, возвращает JSON для API-маршрутов, возвращает HTML для страницы `/html`, поддерживает `POST /echo` для возврата тела запроса и показывает, как один сервер может обслуживать сразу несколько форматов ответа |

---

# 🧠 Возможности

| 🌐 Что умеет сервер                       | 📖 Как это работает                                                                   |
| ----------------------------------------- | ------------------------------------------------------------------------------------- |
| Возвращать JSON для основных маршрутов    | Для путей `/`, `/hello` и `/bye` сервер отправляет JSON-ответы с сообщениями          |
| Возвращать HTML страницу                  | Для пути `/html` сервер отправляет полноценный HTML-ответ с `Content-Type: text/html` |
| Принимать POST запрос с телом             | Для пути `/echo` сервер читает тело запроса и возвращает его в JSON                   |
| Обрабатывать неизвестные пути             | Для неизвестного маршрута сервер возвращает JSON с ошибкой `Not found`                |
| Работать как универсальный учебный сервер | Один и тот же сервер демонстрирует маршрутизацию, JSON API, HTML и обработку body     |

---

# 🌐 Маршруты

| 🌐 Запрос     | 📥 Что возвращает сервер                                                            |
| ------------- | ----------------------------------------------------------------------------------- |
| `GET /`       | Сервер возвращает JSON `{"message":"Main page"}` как ответ для главной страницы     |
| `GET /hello`  | Сервер возвращает JSON `{"message":"Hello!"}` как отдельный API-маршрут             |
| `GET /bye`    | Сервер возвращает JSON `{"message":"Goodbye!"}` как отдельный API-маршрут           |
| `POST /echo`  | Сервер возвращает JSON `{"echo":"..."}` и подставляет туда тело клиентского запроса |
| `GET /html`   | Сервер возвращает HTML страницу с текстом `Hello from HTML page`                    |
| `ANY unknown` | Сервер возвращает JSON `{"error":"Not found"}` для неизвестного пути                |

---

# 🔍 Проверка и результат

| 🔍 Что выполнить                                                                                                                       | 📥 Какой должен быть результат                                                                                                                                |
| -------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `curl http://localhost:8080/ ; curl http://localhost:8080/hello ; curl http://localhost:8080/bye ; curl http://localhost:8080/unknown` | Сервер должен вернуть JSON с сообщениями `Main page`, `Hello!`, `Goodbye!` и JSON с ошибкой `Not found`, что показывает корректную маршрутизацию API-запросов |

---

# 🔁 Echo и HTML

| 🔍 Что выполнить                                                 | 📥 Какой должен быть результат                                                                                                  |
| ---------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `curl -X POST http://localhost:8080/echo -d "Hello from client"` | Сервер должен вернуть JSON `{"echo":"Hello from client"}`, то есть прочитать тело POST-запроса и отправить его обратно в ответе |
| `curl http://localhost:8080/html`                                | Сервер должен вернуть HTML страницу, внутри которой отображается текст `Hello from HTML page`, а не JSON                        |

# 🖥 Логи сервера

| 🖥 Что видно в терминале сервера                                             | 📖 Что это доказывает                                                                                                                             |
| ---------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Connection received, GET /, POST /echo, Body: Hello from client, GET /html` | Сервер принимает соединения, различает HTTP-методы и маршруты, умеет читать тело запроса и формировать разные ответы для JSON API и HTML страницы |

---

# 🌐 HTTP ответ

| 🌐 Что получает клиент                                                                                                          | 📖 Почему это важно                                                                                                                            |
| ------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Для JSON-маршрутов клиент получает JSON с правильным содержимым, а для `/html` клиент получает HTML с правильным `Content-Type` | Это показывает, что сервер умеет не просто отвечать строкой, а осознанно выбирать формат ответа в зависимости от маршрута и назначения ресурса |

# 🧩 Логика обработки

| 🧩 Как работает сервер | 📖 Что происходит внутри                                 |
| ---------------------- | -------------------------------------------------------- |
| Accept соединение      | Сервер принимает входящее TCP-соединение от клиента      |
| Read HTTP request      | Сервер читает HTTP-запрос и извлекает метод, путь и тело |
| Проверка method        | Сервер определяет, какой HTTP-метод был использован      |
| Проверка target        | Сервер определяет, какой маршрут запросил клиент         |
| Выбор формата ответа   | Сервер выбирает JSON или HTML в зависимости от маршрута  |
| Write response         | Сервер отправляет готовый HTTP-ответ обратно клиенту     |

# ⚠️ Важно

| ⚠️ Проблема                             | 📖 Причина и решение                                                          |
| --------------------------------------- | ----------------------------------------------------------------------------- |
| JSON не читается клиентом корректно     | Нужно отдавать правильный `Content-Type: application/json` для JSON-маршрутов |
| HTML отображается как обычный текст     | Нужно отдавать правильный `Content-Type: text/html` для HTML-ответа           |
| Echo не работает                        | Для маршрута `/echo` нужно использовать именно `POST`, а не `GET`             |
| Неизвестный путь ведёт себя неправильно | Для неизвестного маршрута сервер должен возвращать JSON с ошибкой             |
