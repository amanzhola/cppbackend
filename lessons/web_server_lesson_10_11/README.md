# 🚀 Практическое занятие №4

## Изменить тело ответа

---

# 📌 Кратко

| 📌 Что это                                    | 📖 Полное описание                                                                                                                                                                        |
| --------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Учебный HTTP-сервер с изменяемым HTML-ответом | В этом задании сервер возвращает HTML-ответ, а основная цель урока состоит в том, чтобы изменить содержимое `response.body()` и увидеть, как браузер отображает уже другой HTML-результат |

---

# 🎯 Цель

| 🎯 Что нужно понять                                           | 📖 Что это означает на практике                                                                |
| ------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| Изменение body меняет отображение в браузере                  | Если сервер отправляет другой HTML-код, браузер отображает уже другую страницу                 |
| HTML-ответ зависит не только от body, но и от заголовков      | Для корректного отображения сервер должен отправлять `Content-Type: text/html`                 |
| Для этого урока соединение лучше закрывать сразу после ответа | Учебный пример с одним запросом и одним ответом работает надёжнее и не вызывает зависания curl |

---

# ⚙️ Основной код

```cpp
StringResponse res{http::status::ok, req.version()};
res.set(http::field::server, "Lesson10_11");
res.set(http::field::content_type, "text/html; charset=utf-8");
res.keep_alive(false);
res.body() = "<h1>Hello, Beast!</h1>\n";
res.prepare_payload();
```

---

# 🧠 Что было исправлено

| 🧠 Ошибка                                                                                                         | 📖 Исправление                                                                                                                                        |
| ----------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| В предыдущем варианте логика соединения была сделана слишком сложно для этого урока, из-за чего curl мог зависать | Для lesson 10_11 оставлен простой и надёжный шаблон: один запрос, один ответ, `keep_alive(false)`, закрытие соединения и возврат к следующему клиенту |
| Поведение отличалось от уже исправленного lesson 10_10                                                            | Логика 10_11 приведена к тому же безопасному шаблону, который уже сработал в 10_10                                                                    |
| Из-за этого клиент мог ждать закрытия соединения                                                                  | Теперь сервер явно закрывает соединение после отправки HTML-ответа                                                                                    |

---

# 🌐 Что именно проверяется

| 🌐 Элемент                           | 📖 Что должно произойти                                                     |
| ------------------------------------ | --------------------------------------------------------------------------- |
| Тело ответа `<h1>Hello, Beast!</h1>` | Браузер показывает крупный заголовок Hello, Beast!                          |
| Content-Type                         | Сервер отправляет `text/html; charset=utf-8`                                |
| curl                                 | curl показывает HTML-код `<h1>Hello, Beast!</h1>`                           |
| Поведение соединения                 | После ответа сервер закрывает соединение и не зависает на следующем клиенте |

---

# 🔍 Проверка и результат

| 🔍 Что выполнить                                                                                                              | 📥 Какой результат должен быть                                                                                                                                                                |
| ----------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Открыть `http://localhost:8080/` в браузере, затем выполнить `curl http://localhost:8080/` и `curl -i http://localhost:8080/` | В браузере отображается заголовок Hello, Beast!, обычный curl показывает HTML-код, а `curl -i` показывает корректный заголовок `Content-Type: text/html; charset=utf-8` и закрытие соединения |

# ⚠️ Важно

| ⚠️ Проблема                       | 📖 Причина и решение                                                                           |
| --------------------------------- | ---------------------------------------------------------------------------------------------- |
| curl зависает                     | Соединение не закрывается после ответа, для этого урока нужно использовать `keep_alive(false)` |
| Браузер показывает HTML как текст | Неверно указан `Content-Type`, нужно использовать `text/html`                                  |
| Изменения не видны                | Сервер не был пересобран после изменения `main.cpp`                                            |
