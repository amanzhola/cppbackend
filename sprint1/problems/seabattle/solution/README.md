# 🚢 Seabattle (TCP) — Полноценная сетевая игра

## 📌 Описание

Это реализация игры **«Морской бой» по TCP**.

В отличие от предыдущих заданий (например, `radio`), здесь реализовано:

* двустороннее сетевое взаимодействие
* синхронизация состояния игры
* протокол обмена ходами
* полноценная игровая логика

📡 Игра работает по сети:

```text
Client ↔ TCP ↔ Server
```

# ⚠️ КРИТИЧЕСКИЙ БАГ И ЕГО ИСПРАВЛЕНИЕ

## ❌ Ошибка

Изначально координаты интерпретировались неправильно:

```text
буква → x
цифра → y   ❌
```

Это приводило к:

* выстрелы попадали не туда
* поля рассинхронизировались

---

## ✅ Правильное соответствие

```text
буква → строка → y
цифра → столбец → x
```

### Примеры:

```text
A1 → (x=0, y=0)
B6 → (x=5, y=1)
H8 → (x=7, y=7)
```

---

## 🔧 Исправление

### ✔ ParseMove

```cpp
int y = sv[0] - 'A';
int x = sv[1] - '1';
return {x, y};
```

### ✔ MoveToString

```cpp
char('A' + y), char('1' + x)
```

# 🎮 Как играть

После запуска:

```text
Your move (e.g. B6):
```

Ввод:

```text
B6
```

Правила:

* A–H — строки
* 1–8 — столбцы
* ровно 2 символа

---

# 🔍 Что происходит внутри

## При ходе игрока

1. Ввод координаты
2. Отправка 2 байт
3. Получение результата (1 байт)
4. Обновление поля соперника

---

## При ходе соперника

1. Чтение 2 байт
2. Выстрел по своему полю
3. Отправка результата
4. Обновление состояния

# 📡 Почему используется TCP

TCP выбран потому что:

* гарантирует доставку
* сохраняет порядок
* исключает потери

👉 В отличие от UDP (`radio`), здесь критична целостность данных

---

# ⚡ Сравнение с radio

| Характеристика | radio         | seabattle    |

| Протокол       | UDP           | TCP          |
| Направление    | одностороннее | двустороннее |
| Состояние      | нет           | есть         |
| Логика         | поток данных  | игровая      |
