# 🎯 Цель

| 🎯 Что нужно было реализовать    | 📖 Что это означает на практике                                                                                                                                                              | 🔥 На что обращается внимание                                                                      | 🏁 Что получилось                                                                 |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- |
| Ответ на `GET`                   | Сервер должен вернуть статус `200 OK`, заголовок `Content-Type: text/html`, корректный `Content-Length` и тело вида `Hello, {target}`, где из target убирается ведущий символ `/`            | Нужно правильно извлечь путь запроса и построить строку ответа без ошибок в длине тела             | Для запроса `/Ivan` сервер возвращает `Hello, Ivan` и `Content-Length: 11`        |
| Ответ на `HEAD`                  | Сервер должен сформировать такой же ответ, как для `GET`, но без тела, при этом `Content-Length` должен остаться таким же, как у соответствующего `GET`                                      | Важно не перепутать: тело пустое, но длина считается как для полного текста ответа                 | Это реализовано через вычисление длины строки и отсутствие записи body для `HEAD` |
| Ответ на неподдерживаемые методы | Для методов, отличных от `GET` и `HEAD`, сервер должен вернуть `405 Method Not Allowed`, заголовок `Allow: GET, HEAD`, `Content-Type: text/html` и тело `Invalid method` с корректной длиной | Здесь важно не забыть заголовок `Allow` и правильный статус, а также соблюсти ожидаемую длину тела | Сервер отвечает `405` и сообщает клиенту, какие методы разрешены                  |
| Готовность сервера к тестам      | После запуска сервер должен вывести в stdout строку `Server has started...`, чтобы тесты начали работу только после старта сервера                                                           | Это обязательная деталь для автоматической проверки                                                | Строка выводится до входа в цикл `accept`                                         |

# ⚙️ Основной код

```cpp
StringResponse HandleRequest(StringRequest&& req) {
    if (req.method() != http::verb::get && req.method() != http::verb::head) {
        return MakeMethodNotAllowedResponse(req.version(), req.keep_alive());
    }

    std::string name = GetTarget(req.target());
    std::string body = "Hello, " + name;

    StringResponse res(http::status::ok, req.version());
    res.set(http::field::content_type, ContentType::TEXT_HTML);
    res.content_length(body.size());
    res.keep_alive(req.keep_alive());

    if (req.method() == http::verb::get) {
        res.body() = body;
    }

    return res;
}
```

# 🧠 Главная идея

| 🧠 Ключевой шаг           | 📖 Что происходит                                                                                 | ⚠️ Что важно не сломать                                                                                | ✅ Почему это правильно                                                 |
| ------------------------- | ------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------- |
| Проверка HTTP-метода      | Сервер сначала смотрит, пришёл ли `GET`, `HEAD` или другой метод                                  | Если не обработать это явно, сервер не сможет вернуть правильный статус `405` и заголовок `Allow`      | Явная проверка метода делает поведение сервера полностью предсказуемым |
| Извлечение target         | Из пути запроса убирается первый символ `/`, после чего формируется строка `Hello, {target}`      | Нужно убрать только ведущий `/`, а не испортить всю строку                                             | Для `/Ivan` получается `Ivan`, а итоговое тело — `Hello, Ivan`         |
| Разделение `GET` и `HEAD` | Для `GET` тело реально записывается, а для `HEAD` не записывается, но длина выставляется такой же | Это одна из самых частых ошибок: забыть, что `HEAD` возвращает пустое тело, но тот же `Content-Length` | Реализация точно соответствует HTTP-смыслу задания                     |
| Ответ на неверный метод   | Для неподдерживаемых методов сервер возвращает отдельный готовый response с `405` и `Allow`       | Нельзя забыть ни заголовок, ни длину тела, ни `Content-Type`                                           | Именно так и ожидают тесты                                             |

# 🌐 HTTP смысл

| 🌐 Что важно в HTTP                                      | 📖 Как это реализовано в решении                                                                            | ⚠️ Почему это принципиально                             | ✅ Практический вывод                                               |
| -------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- | ------------------------------------------------------- | ------------------------------------------------------------------ |
| `Content-Length` должен совпадать с реальной длиной тела | Для `GET` длина считается по строке `Hello, {target}`, для `HEAD` — по той же строке, даже если тело пустое | Именно по этому заголовку клиент понимает размер ответа | Нельзя добавлять лишний символ `\n`, если этого не требует условие |
| `HEAD` похож на `GET`, но без тела                       | Сервер создаёт тот же ответ по смыслу, но body не заполняет                                                 | Это базовое и важное правило HTTP                       | Такая реализация проходит проверку и соответствует протоколу       |
| `405 Method Not Allowed` должен сопровождаться `Allow`   | Для неподдерживаемых методов сервер явно сообщает, что разрешены только `GET, HEAD`                         | Это не косметика, а часть корректного HTTP-поведения    | Клиент видит не просто ошибку, а список допустимых методов         |
| `Content-Type` должен быть точным                        | В ответах используется `text/html`                                                                          | Автотесты часто проверяют это значение буквально        | Точное значение заголовка здесь обязательно                        |

# ⚠️ Важное замечание про перенос строки

| ⚠️ Что может захотеться сделать | 📖 Почему так делать не нужно                                                                                                               | 🔍 Что делать вместо этого                                                                                              | ✅ Правильный вариант                                                                                  |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| Добавить `\n` в тело ответа     | Тогда строка `Hello, Ivan` превратится в `Hello, Ivan\n`, а `Content-Length` изменится с `11` на `12`, что уже противоречит условию задания | Если хочется красивого отображения в терминале, добавлять перевод строки нужно не в сервер, а снаружи при вызове `curl` | Использовать `curl http://127.0.0.1:8080/Ivan; echo` или `curl -i http://127.0.0.1:8080/Ivan && echo` |

| ---------------------------------------------------- | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------- | -------------------------------------------------------- |
| Сервер корректно отвечает на `GET`                   | Возвращается `200 OK`, `Content-Type: text/html`, правильный `Content-Length` и тело `Hello, {target}` | Неверная длина, отсутствие заголовка или неправильное тело | Ответ полностью совпадает с условием                     |
