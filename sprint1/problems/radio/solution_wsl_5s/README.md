# 📡 UDP Radio (WSL ↔ Mac) — передача аудио

---

## 📌 Описание

В этом проекте реализована передача аудио по UDP:

* 🎤 запись звука на клиенте (Mac)
* 📦 разбиение на чанки
* 🌐 отправка по сети
* 📥 приём в WSL (Windows)
* 🔊 воспроизведение звука

---

## 🎯 Цель

Понять и реализовать:

* работу UDP
* собственный протокол передачи данных
* передачу бинарных данных (аудио)
* взаимодействие между разными системами (Mac ↔ Windows через WSL)

---

# 🧨 Проблема (почему была "тишина")

Сервер **не принимал данные**, хотя:

* сеть работала ✅
* ping проходил ✅
* UDP-пакеты отправлялись ✅

---

## 💥 Главная ошибка

Клиент отправлял:

```bash
echo ping | nc -u 192.168.0.177 3333
```

👉 Это просто:

```text
"ping" = 5 байт
```

---

## ❗ Но сервер ожидал

Структуру:

```cpp
struct PacketHeader {
    uint32_t magic;
    uint32_t total_size;
    uint16_t chunk_index;
    uint16_t chunk_count;
    uint16_t payload_size;
};
```

---

## 🧠 Итог

```text
nc → НЕ отправляет header
сервер → игнорирует пакет

👉 поэтому ТИШИНА
```

---

# ⚠️ Вторая проблема (критическая)

Сервер начинал сборку только если:

```cpp
chunk_index == 0
```

---

## ❗ Что это означает

Если первый чанк потерян (а UDP это допускает):

```text
chunk 1, 2, 3...
НО НЕТ chunk 0
```

👉 сообщение никогда не собирается

---

# 🧨 Третья проблема

```cpp
received_chunks++
```

Без проверки:

* дублей ❌
* порядка ❌
* уже полученных чанков ❌

---

# 🧠 Вывод

```text
Проблема была НЕ в сети
Проблема была в ПРОТОКОЛЕ
```

---

# 🔧 Решение

Был полностью переработан протокол передачи данных.

---

## 📦 Новый заголовок пакета

```cpp
struct PacketHeader {
    uint32_t magic;
    uint32_t message_id;
    uint32_t total_size;
    uint16_t chunk_index;
    uint16_t chunk_count;
    uint16_t payload_size;
};
```

---

## 🔥 Что добавлено

### ✅ message_id

Позволяет:

* обрабатывать несколько сообщений
* не смешивать пакеты

---

### ✅ Приём чанков в любом порядке

Теперь сервер:

* НЕ зависит от `chunk 0`
* собирает данные по индексам

---

### ✅ Защита от дублей

```text
если chunk уже получен → игнорируется
```

---

### ✅ Проверка валидности пакета

* magic number
* размеры
* границы данных

---

### ✅ Очистка старых сообщений

Если сообщение не собрано за время:

```text
→ удаляется (TTL)
```
